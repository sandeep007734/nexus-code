IBM AFS Windows 版快速入门

介绍

AFS(R) 是一个通过共享文件空间提供一致文件存取的企业文件系统。 通过将多个文件服务器机器的本地文件系统连接在一起，AFS 展现了一种独立于机器界限的单个文件空间。 文件存储在计算机网络中的各个不同机器上，但企业中的所有机器都可存取它们。

IBM AFS Windows 版(R)，版本 3.6 将 AFS 的全部功能都延伸到了 Microsoft(R) Windows 操作系统上。

文档概述

本文档概述安装的先决条件， 提供有关安装、配置和卸装 AFS Windows 版的详细指示信息，略述安装和卸装进程期间对系统的更改。 还描述 AFS Windows 版提供的文档。

读者

本文档为负责 AFS Windows 版中产品安装和配置的系统管理员和用户提供信息。 假设系统管理员熟悉一般的系统管理，用户熟悉 Microsoft Windows 操作系统的基本术语和概念。

组织

本文档结构如下：

  • 安装 AFS Windows 版略述可安装的 AFS 组件组合， 描述在 AFS Windows 版中安装产品的步骤，并列出安装进程中对系统的更改。

  • AFS Windows 版文档展示 AFS Windows 版提供的各种类型文档， 并讨论访问这些文档的步骤。

  • 配置 AFS Windows 版描述在 AFS Windows 版中配置产品的步骤。

  • 卸装 AFS Windows 版略述卸装先决条件， 描述卸装包含在 AFS Windows 版中产品的步骤，并列出卸装进程中对系统的更改。

安装 AFS Windows 版

本节略述可安装的 AFS 组件组合，描述安装 AFS Windows 版的步骤，并列出安装进程对系统的更改。

AFS Windows 版组件

AFS Windows 版，版本 3.6 包含下列组件：

  • AFS 服务器

    AFS 服务器运行 AFS 服务器进程，并包含 AFS 服务器配置向导以帮助设置。

  • AFS 控制中心

    AFS 控制中心包含两个功效强大的图形用户界面（GUI）工具：AFS 服务器管理器和 AFS 帐户管理器，用在 AFS 单元管理方面给 AFS 系统管理员提供帮助。

  • AFS 客户机

    AFS 客户机提供对 AFS 文件空间的直接存取，使用户能对 AFS 中的文件和目录进行管理。 AFS 客户机包含 AFS Light 网关。

  • AFS Light

    AFS Light 通过一个 AFS Light 网关机器提供对 AFS 文件空间的存取，使用户能对 AFS 中的文件和目录进行管理。

  • AFS 补充文档

    AFS 补充文档提供其它 AFS 系统管理信息，并包含下列文档：IBM AFS Administration Guide 和 IBM AFS Administration Reference。

安装选项

根据您的 Windows 操作系统， 您可以各种组合来安装 AFS Windows 版的各个组件。 请参考 IBM AFS Windows 版发行注意事项，了解对于每个 AFS Windows 版组件的特定软件要求的详细信息。 请注意，如果您要安装 AFS 服务器，则必须同时安装 AFS 客户机，除非 AFS 客户机，版本 3.6 已在机器上安装。 无论您要安装哪些组件，都请遵循安装 AFS Windows 版中描述的安装步骤。
注: 您可改动 AFS Windows 版安装程序，以禁用除客户机外的所有其它组件。 此种纯客户机安装程序会使用户无法安装除 AFS 客户机外的所有其它组件。 要执行纯客户机安装，在存放其它安装文件的同一目录中创建文件 setup.co； 安装程序将只允许安装 AFS 客户机。注意，setup.co 文件的内容是不相关的。 无论您要执行何种类型的安装，都请遵循安装 AFS Windows 版中描述的安装步骤。

从先前版本升级

在 Windows NT 机器上，无须为升级软件而卸装 AFS Windows 版的组件； 您可在您的系统上安装此发行版的 AFS Windows 版，而无须除去或解除您现有软件的配置。 要升级 AFS Windows 版，请遵循安装 AFS Windows 版中描述的安装步骤。 在安装进程期间，先前版本的 AFS 组件被升级，而 AFS 配置信息被保留。

在 Windows 95 或 Windows 98 机器上，升级 AFS Light 之前，您必须卸装先前安装的 AFS Light 组件，如卸装 AFS Windows 版中所述。

注意，AFS Windows 版安装工具不允许用户安装具有不同版本号的 AFS 组件。 此外，如果您的机器上安装有一个以上 AFS Windows 版组件，则您不能只更新某一个组件而不更新其它所有的组件。

安装 AFS Windows 版

在安装 AFS Windows 版之前，请参考 IBM AFS Windows 版发行注意事项，以了解安装先决条件的详细说明。 如果您正在运行其它 Windows 应用程序，则建议您在安装 AFS Windows 版之前退出。

  1. 将 AFS Windows 版安装盘插入 CD-ROM 驱动器中。

  2. 通过使用以下方法之一运行 AFS Windows 版 setup 程序：

    • 从开始菜单，选择运行。 输入 drive:\setup， 其中 drive 是 CD-ROM 驱动器的盘符。 单击确定。

    • 在 Windows 资源管理器中，选择 CD-ROM 驱动器，再双击 setup.exe 程序。

  3. 欢迎对话框出现。选择下一个按钮，继续安装进程。

  4. 选择组件对话框出现。

    • 在组件框中，选择您要安装或升级的 AFS Windows 版组件。 请参阅安装选项，了解有关可在 Windows 机器上安装的各种组件组合。 请注意，如果您要安装 AFS 服务器，则必须同时选择安装（或升级）AFS 客户机，除非 AFS 客户机，版本 3.6 已在系统上安装。

    • 目标文件夹框显示安装选定组件的缺省驱动器和目录。 缺省驱动器是安装 Windows 的驱动器。驱动器上的缺省目录是 \Program Files\Ibm\Afs。 要选择另一个驱动器和/或目录，可选择浏览按钮。
      注:如果您从一个先前版本的 AFS Windows 版升级或重新安装 AFS Windows 版， 则您选择的安装目录必须与先前安装版本使用的安装目录相同。

    选择下一个按钮，继续安装进程。

  5. 选定 AFS Windows 版组件的应用程序文件安装到您的系统上。 当安装进程结束时，安装完成对话框将出现，指示在可使用已安装 AFS 产品前必须重新启动系统。 选择是，现在重新启动计算机，然后选择结束按钮。 您的系统将关机，然后再重新启动。

    AFS Windows 版的安装已完成。

对系统的更改

本节描述安装每个 AFS Windows 版组件对系统的更改。 本节中的信息基于缺省安装设置。

安装 AFS 客户机对系统的更改

安装 AFS 客户机 Windows NT 版对您的系统有下列更改：

  • 创建一个名为 IBM AFS 的开始菜单程序组，包含下列应用程序：

    文档程序条目允许访问 AFS Windows 版提供的 AFS 联机文档集。

    客户机程序子组允许用户访问“AFS 客户机”特性框和 AFS 客户机联机帮助。

  • 将 AFS 菜单添加到 Windows NT 资源管理器的上下文菜单中。

  • 创建一个文档目录（\Program Files\Ibm\Afs\Documentation）， 并存放 IBM AFS Windows 版快速入门和 IBM AFS Windows 版发行注意事项联机文档。

  • 将 AFS 凭证添加到启动程序组中。 AFS 客户机图标会在启动时显示在任务栏中。

  • 创建安装程序用于安装 AFS 二进制文件、图标和帮助文件的安装目录。 缺省目录是 \Program Files\Ibm\Afs\Client\Program 和 \Program Files\Ibm\Afs\Common。

  • 将 AFS Client（AFS 客户机）注册为一项服务。

  • 安装 AFS 客户机配置实用程序，并通过将 afs_cpa.cpl 文件放入 \WindowsDefault\system32 目录（其中 WindowsDefault 是您的 Windows 目录）， 将 AFS 客户机配置图标添加到控制面板中。

  • 将 afsdcell.ini 文件放入您的 Windows 目录和 \Program Files\Ibm\Afs\Common 目录。 如果您从一个先前版本的 AFS 客户机升级，则不替换 Windows 目录中的 AFS 客户机单元数据库（afsdcell.ini）。

  • 修改 Windows NT 注册表，添加与 AFS 客户机相关的条目。

安装 AFS Light 对系统的更改

安装 AFS Light Windows 95 和 Windows 98 版对您的系统有下列更改：

  • 创建一个名为 IBM AFS 的程序组，包含下列应用程序：

    文档程序条目允许访问 AFS Windows 版提供的 AFS 联机文档集。

    Light 程序子组允许用户访问“AFS Light”特性框和 AFS Light 联机帮助。

  • 将 AFS 菜单添加到 Windows 资源管理器的上下文菜单中。

  • 创建一个文档目录（\Program Files\Ibm\Afs\Documentation）， 并存放 IBM AFS Windows 版快速入门和 IBM AFS Windows 版发行注意事项联机文档。

  • 创建安装程序用于安装 AFS 二进制文件、图标和帮助文件的安装目录。 缺省目录是 \Program Files\Ibm\Afs\Client\Program 和 \Program Files\Ibm\Afs\Common。

  • 安装 AFS Light 配置实用程序，并通过将 afs_cpa.cpl 文件放入 \WindowsDefault\system 目录（其中 WindowsDefault 是您的 Windows 目录）， 将 AFS Light 配置图标添加到控制面板中。

  • 将 afsdcell.ini 文件放入您的 Windows 目录和 \Program Files\Ibm\Afs\Common 目录。

  • 修改 Windows 注册表，添加与 AFS Light 相关的条目。

安装 AFS 服务器对系统的更改

安装 AFS 服务器 Windows NT 版对您的系统有下列更改：

  • 创建一个名为 IBM AFS 的开始菜单程序组，包含下列应用程序：

    文档程序条目允许访问 AFS Windows 版提供的 AFS 联机文档集。

    服务器程序子组使用户能访问 AFS 服务器快速启动向导。

  • 创建一个文档目录（\Program Files\Ibm\Afs\Documentation）， 并存放 IBM AFS Windows 版快速入门和 IBM AFS Windows 版发行注意事项联机文档。

  • 创建安装程序用于安装 AFS 二进制文件、图标和帮助文件的安装目录。 缺省目录是 \Program Files\Ibm\Afs\Server\usr\afs\bin 和 \Program Files\Ibm\Afs\Common。

  • 将 AFS Server（AFS 服务器）注册为一项服务。

  • 安装 AFS 服务器配置应用程序，并通过将 afsserver_cpa.cpl 文件放入 \WindowsDefault\system32 目录（其中 WindowsDefault 是您的 Windows 目录）， 将 AFS 服务器配置图标添加到控制面板中。

  • 修改 Windows NT 注册表，添加与 AFS 服务器相关的条目。

安装 AFS 控制中心对系统的更改

安装 AFS 控制中心 Windows NT 版对您的系统有下列更改：

  • 创建一个名为 IBM AFS 的开始菜单程序组，包含下列应用程序：

    文档程序条目允许访问 AFS Windows 版提供的 AFS 联机文档集。

    控制中心程序子组使用户能访问帐户管理器和服务器管理器。

  • 创建一个文档目录（\Program Files\Ibm\Afs\Documentation）， 并存放 IBM AFS Windows 版快速入门和 IBM AFS Windows 版发行注意事项联机文档。

  • 创建安装程序用于安装 AFS 二进制文件、图标和帮助文件的安装目录。 缺省目录是 \Program Files\Ibm\Afs\Control Center 和 \Program Files\Ibm\Afs\Common。

安装 AFS 补充文档对系统的更改

安装 AFS 补充文档对您的系统有下列更改：

  • 创建一个名为 IBM AFS 的开始菜单程序组，包含文档程序条目。

  • 在机器上安装下列系统管理文档：IBM AFS Administration Guide 和 IBM AFS Administration Reference。 这些文档被添加到联机文档目录（\Program Files\Ibm\Afs\Documentation）中。 IBM AFS Windows 版快速入门和 IBM AFS Windows 版发行注意事项联机文档也安装在文档目录中。

  • 修改 Windows NT 注册表，添加与 AFS 补充文档相关的条目。

AFS Windows 版文档

本节描述 AFS Windows 版提供的文档，并细化访问这些文档的步骤。

联机文档目录

无论您在系统上安装了哪些组件，文档目录总会创建。 缺省位置是 \Program Files\Ibm\Afs\Documentation。 此目录包含 IBM AFS Windows 版快速入门和 IBM AFS Windows 版发行注意事项。 这些文档也可通过访问启动菜单中文档条目的“文档”索引得到。

如果您安装了 AFS 补充文档，则文档目录还包含下列文档：IBM AFS Administration Guide 和 IBM AFS Administration Reference。 这些文档也可通过访问启动菜单中文档条目的“文档”索引得到。

要访问联机文档目录：

  1. 在开始菜单，选择程序，然后选择 IBM AFS，再选择文档。

  2. 选择您要查看的文档。

CD-ROM 文档目录

AFS Windows 版 CD-ROM 包含一个文档目录。 此目录包含下列文档：IBM AFS Windows 版快速入门、IBM AFS Windows 版发行注意事项、IBM AFS Administration Guide 和 IBM AFS Administration Reference。 文档以 HTML 和 PDF 格式提供。

要访问 CD-ROM 文档目录：

  1. 将 AFS Windows 版 CD-ROM 插入机器的 CD-ROM 驱动器中。

  2. 按照下面列出的路径之一执行。注意，CD 是 CD-ROM 驱动器的盘符。

    • 对于 HTML 文档，按照路径 CD:\Documentation\Html。

    • 对于 PDF 文档，按照路径 CD:\Documentation\Pdf。

联机帮助

联机帮助是与每个 AFS Windows 版组件一起安装的。 联机帮助文档描述每个组件中的可用功能。 使用帮助菜单和位于绝大多数对话框中的帮助按钮来访问联机帮助。 您可通过浏览目录页面获得有关主题的帮助，使用索引定位主题，以及使用查找（联机帮助搜索引擎）。

配置 AFS Windows 版

本节详细描述每个 AFS Windows 版组件的配置步骤。 在您可使用 AFS 前，必须在系统上配置各个组件。

配置 AFS 客户机

注:如果您要在 Windows NT 系统上配置 AFS 服务器，则不需要配置 AFS 客户机。 当 AFS 服务器配置时，AFS 客户机会自动配置。 此外，如果您从一个先前版本的 AFS 客户机升级到此版本的 AFS Windows 版，则配置信息将保留。 您不需要重新配置 AFS 客户机。

  1. 在开始菜单中，选择设置，然后选择控制面板。

  2. 双击 AFS 客户机配置图标。 AFS 客户机配置实用程序打开，显示一般标签。

  3. 在单元名框中，输入一个 AFS 单元的名称，您的机器将成为该单元中的一个客户机。

  4. 选择 AFS 单元标签。 如果此 AFS 单元（机器将成为其中的一个客户机）没有列出在 AFS 单元的列表中，则选择添加按钮。 “新建单元”对话框打开。在 AFS 单元框中输入单元名，在描述框中输入一个简短的说明。

    选择添加按钮。 “添加服务器”对话框打开。在服务器名框中，输入所选单元中某一卷位置服务器的名称。 选择确定以关闭“添加服务器”对话框。 重复此过程，为单元中的所有卷位置服务器添加信息。 （如果您不知道 AFS 单元中卷位置服务器的名称，请向您的 AFS 系统管理员咨询。） 在输入了所有服务器的信息后，选择确定以关闭“新建单元”对话框。

  5. 选择一般标签，并选择启动服务按钮以启动 AFS Client（AFS 客户机）服务。

  6. 选择盘符标签。 要将 Windows NT 机器上的一个盘符映射到 AFS 文件空间，选择添加按钮。 “映射盘符”对话框打开。

  7. 在盘符框中，选择要映射到 AFS 文件空间的驱动器，或接受缺省值。 在 AFS 路径框中，指明您要将选定驱动器映射到的 AFS 位置，例如 /afs。 如果需要，在描述框中输入 AFS 驱动器映射的说明。 选择确定，将驱动器连接到 AFS 文件空间中的指定放置。

  8. 选择确定以关闭“AFS 客户机配置”实用程序。

    现在 AFS 客户机已在所选 AFS 单元中配置，并且 AFS 文件空间可在 Windows NT 资源管理器中通过所选的驱动器映射进行访问。

将 AFS 客户机配置为 AFS Light 网关

您可以在 Windows NT 机器上配置 AFS 客户机作为一个 AFS Light 网关使用。 您的 AFS 客户机（配置为 AFS Light 网关）使得 AFS Light 用户可以访问 AFS 文件空间。

  1. 按配置 AFS 客户机中的详细信息，配置 AFS 客户机。

  2. 在开始菜单中，选择设置，然后选择控制面板。

  3. 双击 AFS 客户机配置图标。 AFS 客户机配置实用程序打开，显示一般标签。

  4. 选择提供一个 AFS Light 网关选项。

  5. 选择确定。

    • 如果 AFS Client（AFS 客户机）服务正在运行，则会出现一个消息框，通知您必须重新启动该服务。 选择是以重新启动 AFS Client（AFS 客户机）服务，并启用 AFS Light 网关。

    • 如果 AFS Client（AFS 客户机）服务是停止的，则会出现一个消息框，通知您必须启动 AFS Client（AFS 客户机）服务。 选择是以启动 AFS Client（AFS 客户机）服务，并启用 AFS Light 网关。

  6. 将单元条目添加到 AFS Light 网关的单元数据库中。 注意，为了使 AFS Light 用户可以访问单元，在 AFS Light 单元数据库和 AFS Light 网关单元数据库中都必须有该单元的条目存在。 网关机器的单元数据库中关于单元的信息不正确或缺少，将导致您的 Light 客户机无法存取文件。

    要添加一个条目到单元数据库中：

    访问“AFS Light 配置”实用程序中的 AFS 单元标签，并选择添加按钮。 “新建单元”对话框打开。在 AFS 单元框中输入单元名，在描述框中输入一个简短的说明。

    选择添加按钮。“添加服务器”对话框打开。 在服务器名框中，输入所选单元中某一卷位置服务器的名称。 选择确定以关闭“添加服务器”对话框。 重复此过程，为单元中的所有卷位置服务器添加信息。 （如果您不知道 AFS 单元中卷位置服务器的名称，请向您的 AFS 系统管理员咨询。） 在输入了所有服务器的信息后，选择确定以关闭“新建单元”对话框。

Windows NT 机器现在已配置为一个 AFS Light 网关。 一旦配置为一个 AFS Light 网关，您的 AFS 客户机必须能够在 Windows 环境中认证 AFS Light 用户。 可以通过域用户帐户或通过同步的机器用户帐户来实现该认证。 域用户帐户是在 Windows 域中的用户帐户。 机器用户帐户是仅在特定主机上有效的用户帐户。

当 AFS Light 网关被配置到一个 Windows 域中时，AFS Light 用户必须以网关所属的域中的一个域用户帐户登录， 或以一个（与网关域中的某一域用户帐户具有相同用户名和口令的）机器用户帐户登录。

如果使用机器用户帐户，那么这些帐户必须在 AFS Light 网关和 AFS Light 主机上同步。 用户必须以与（在 AFS Light 网关机器上定义的）某一机器用户帐户相同的用户名和口令登录到 AFS Light 机器上。

配置 AFS Light

AFS Light 通过一个 AFS Light 网关访问 AFS 文件空间。 在配置 AFS Light 之前，您必须有一台运行 AFS 客户机并配置为 AFS Light 网关的 Windows NT 机器。 如需更多信息，请参阅将 AFS 客户机配置为 AFS Light 网关。

  1. 在开始菜单中，选择设置，然后选择控制面板。

  2. 双击 AFS Light 配置图标。 “AFS Light 配置”实用程序打开，显示一般标签。

  3. 在网关框中，输入要配置为 AFS Light 网关的 Windows NT 机器的名称，并单击现在连接。 网关机器的名称是网关的 NetBIOS 服务名，格式为 mach-afs，其中 mach 是最多为 11 个字符的主机名。 为了与网关机器进行通信，AFS Light 必须能够分辨该服务名。 通过将网关的 NetBIOS 服务名添加到客户机的 LMHOSTS 文件或相应的 DNS 或 WINS 服务器中来实现名称解析。 如果 AFS Light 机器和它的 AFS Light 网关机器在同一子网中，则不需要其它的配置。

    AFS Light 自动成为与其 AFS Light Gateway 相同的单元中的一个成员。 单元的名称显示在单元名框中。

    注:如果 AFS Light 网关机器和 AFS Light 机器位于同一域中，并且此域中网关机器的主机名是 xyz-pc， 则您可在网关框中将计算机名指定为 xyz-pc 或 xyz-pc.xcompany.com。

  4. 选择 AFS 单元标签。 如果机器所属的单元没有列出在 AFS 单元的列表中，则选择添加按钮。 “新建单元”对话框打开。在 AFS 单元框中输入单元名，在描述框中输入一个简短的说明。

    选择添加按钮。 “添加服务器”对话框打开。在服务器名框中，输入所选单元中某一卷位置服务器的名称。 选择确定以关闭“添加服务器”对话框。 重复此过程，为单元中的所有卷位置服务器添加信息。 （如果您不知道 AFS 单元中卷位置服务器的名称，请向您的 AFS 系统管理员咨询。） 在输入了所有服务器的信息后，选择确定以关闭“新建单元”对话框。

    注意，在 AFS Light 网关的单元数据库（afsdcell.ini 文件）中必须存在一个同样的条目，以便 AFS Light 用户认证到单元。 如需有关同步网关机器单元数据库和您的 Light 客户机单元数据库的更多信息，请参阅将 AFS 客户机配置为 AFS Light 网关。

  5. 选择盘符标签。 要将 Windows 机器上的一个盘符映射到 AFS 文件空间，选择添加按钮。 “映射盘符”对话框打开。

  6. 在盘符框中，选择要映射到 AFS 文件空间的驱动器，或接受缺省值。 在 AFS 路径框中，指明您要将选定驱动器映射到的 AFS 位置，例如 /afs。 如果需要，在描述框中输入 AFS 驱动器映射的说明。 选择确定，将驱动器连接到 AFS 文件空间中的指定放置。

  7. 选择确定以关闭“AFS Light 配置”实用程序。

    现在 AFS Light 已在所选 AFS 单元中配置，并且 AFS 文件空间可在 Windows 资源管理器中通过所选的驱动器映射进行访问。

配置 AFS 服务器

配置过程启动运行 AFS 服务器所需的服务，并在您的 Windows NT 机器上设置 AFS 分区。 使用 AFS 配置向导，您可以快速地将 AFS 服务器配置为新单元中的第一个服务器，或配置为现有单元中的一个服务器。 注意，如果您已经升级到 AFS 服务器当前版本，则先前版本的配置信息仍保留； 您不需要重新配置服务器。

要将 AFS 服务器配置为单元中的第一个 AFS 服务器：

  1. 在开始菜单，选择程序，然后选择 IBM AFS，再选择服务器，再选择配置向导。 “AFS 服务器快速启动向导”打开。

  2. 选择下一步按钮。 “单元和服务器信息”对话框出现。

  3. 选择这将是新单元中的第一个服务器选项。

  4. 在单元名框中输入新 AFS 单元名。

    以下是对可作为 AFS 单元名的 Internet 域名形式的约束：

    • 单元名必须是唯一的，以便将您的 AFS 单元与 AFS 全球名称空间中的其它名称区分开来。

    • 单元名最多可包含 64 个字符；但是，建议使用较短的名称。

    • 单元名必须仅包括小写字符、数字、下划线、破折号和点号以保证在不同操作系统间的可移植性。

    • 单元名可以包括任何数字或字母，通常是用点号分隔。

    • 单元名必须以表明其所属机构的后缀结尾。 一些标准后缀是：用于商业和其它贸易机构的 .com，用于教育机构诸如大学的 .edu，用于政府机构的 .gov 和用于军事机构的 .mil。

  5. 在口令框中，输入字符串作为单元中 AFS 服务器委托人帐户（afs）的口令。 所有的 AFS 服务器象此委托人一样获取 AFS 令牌，并且认证服务器的存取权限授予服务（TGS）模块使用此口令对（在服务器和客户机相互认证时，AFS 客户机提供给服务器的）服务器存取权限进行加密。

  6. 在验证口令框中，重新输入该单元的 AFS 服务器委托人帐户的初始 AFS 口令，以确认所选的口令。

  7. 选择下一步按钮。 “管理信息”对话框出现。

  8. 在名称框中，输入一个用户名作为该单元的通用 AFS 管理帐户（通常是，admin）。

    使用通用的管理帐户就意味您无须授予每个系统管理员特权。 相反，每个管理员知道通用管理员帐户的名称和口令，并在执行需要管理特权的任务时用此身份认证到 AFS 中。

  9. 在口令框中输入一个字符串作为 AFS 管理帐户的口令。

  10. 在验证口令框中重新输入 AFS 管理帐户的口令以确认所选的口令。

  11. 指定 AFS 用户标识符（UID）来分配 AFS 管理帐户：

    • （建议）要自动为 AFS 管理帐户分配下一个可用的 UID ，选择使用下一可用 AFS UID 选项。

    • 要为 AFS 管理帐户指定一个特定 UID ，选择使用该 AFS ID 选项，并在输入框中输入希望的 UID。

      注:通常建议您不要指定一个特定的 UID 给新 AFS 用户帐户，除非您要使该 AFS UID 与现有 UNIX UID 匹配。

  12. 选择下一步按钮。 “文件服务”对话框出现。

    AFS 文件服务器把被请求的文件和数据从服务器发送至 AFS 客户机。 文件服务器存储文件和数据，处理复制、移动、创建和删除文件和目录的请求，并跟踪服务器上每个文件和目录的状态信息。

    因为您正在配置新单元中的第一个 AFS 服务器，所以必须在服务器上配置文件服务，文件服务会自动配置。

  13. 选择下一步按钮。 “数据库服务”对话框出现。

    每个 AFS 单元必须至少包含一个数据库服务器。 每个数据库服务器都运行数据库进程，以维护 AFS 数据库：认证数据库、保护数据库、卷定位数据库和可选的备份数据库。

    因为您正在配置新单元中的第一个 AFS 服务器，所以必须在服务器上配置数据库服务，数据库服务会自动配置。

  14. 选择下一步按钮。 “备份服务器”对话框出现。

    备份服务器维护一个备份数据库，在其中存储与备份系统相关的信息。 备份服务器使 AFS 系统管理员能够将 AFS 文件空间中的数据从卷备份到磁带中。 这样，当文件系统中的数据丢失时（例如，如果由于系统或磁盘故障而引起数据丢失），数据能够从磁带恢复。

  15. 如果要将此 AFS 服务器配置为一个备份服务器，选择是，配置为备份服务器选项。 如果您不想将此 AFS 服务器配置为一个备份服务器，选择否，不配置为备份服务器选项。

    注:如果单元中的任何一个数据库服务器配置了备份服务器，则单元中的所有数据库服务器也必须配置。

  16. 选择下一步按钮。 “AFS 分区”对话框出现。

    每个 AFS 文件服务器必须至少有一个专门指定为存储 AFS 卷的分区，并且所有的 AFS 卷必须驻留在指定为 AFS 分区的分区上。 在 Windows NT 机器上，只有 NTFS 卷能被指定为 AFS 分区。 另外，AFS 分区只能创建在空的 NTFS 卷上（或仅包含 Windows NT 回收站）。

    因为您正在配置新单元中的第一个 AFS 服务器，所以必须在服务器上指定一个 AFS 分区。

  17. 在 NTFS 卷列表中，选择您要指定为 AFS 分区的卷。 在 AFS 分区名框中，输入分区名的最后部分。

    注:在一个 AFS 服务器上最多可存在 256 个 AFS 分区。 约定情况下，每个分区被命名为 /vicepx， 其中 x 是一个或两个小写英文字母。 AFS 分区可命名为 /vicepa、/vicepb、等等，直到 /vicepz。 其它的分区可命名为 /vicepaa 到 vicepaz，等等，直到 /vicepiv。

    极力推荐您使用 NTFS 卷盘符作为分区名的最后一个字母。

  18. 选择下一步按钮。 “根目录 AFS 卷”对话框出现。

    根目录 AFS 卷是两个卷，每个 AFS 单元必须在其文件系统中包括这两个卷。 它们被命名为：

    • root.afs，用于与 AFS 文件空间的顶层（/afs）相对应的卷

    • root.cell，用于安装在 /afs 目录下单元名称位置的卷（例如，yourcompany.com单元中的 /afs/yourcompany.com）

    因为您正在配置新单元中的第一个 AFS 服务器，所以必须在服务器上创建单元的各个根目录卷，而这些卷会在服务器配置期间自动创建。

  19. 选择下一步按钮。 “复制”对话框出现。

    如果您希望利用 AFS 的复制功能，则必须复制 AFS 根目录卷。 复制进程创建 AFS 卷的一个或多个只读副本，并将这些副本分布至一个或多个其它站点（AFS 分区和服务器）。 复制操作使一个 AFS 卷的内容可在一个或多个 AFS 文件服务器机器上存取到，就提高了系统的效率并改进了数据的可用性。

    因为您正在配置新单元中的第一个 AFS 服务器，所以必须在服务器上复制单元的各个根目录卷，而这些卷会在服务器配置期间自动复制。

  20. 选择下一步按钮。 “系统控制服务”对话框出现。

    在运行美国国内版 AFS Windows 版的单元中，系统控制服务器将新版的 AFS 服务器配置信息分布至所有的 AFS 文件服务器。 通常建议您将 AFS 单元中的第一个服务器指定为系统控制服务器。 （运行国际版 AFS Windows 版的单元不使用系统控制服务器来分布系统配置文件。）
    注:如果需要，可在以后将另一服务器机器指定为系统控制服务器。 然而，取决于单元中 AFS 服务器的数量，指定另一控制服务器的进程可能是十分耗时的。

  21. 要将此 AFS 服务器配置为 AFS 单元的系统控制服务器，选择配置为系统控制服务器选项。 如果您不想将此 AFS 服务器配置为 AFS 单元的系统控制服务器，选择不配置为系统控制服务器选项。

  22. 选择下一步按钮。 “配置系统”对话框出现。

    显示配置此 AFS 服务器所要执行步骤的列表，使您可在开始实际的配置进程前复查每一步。
    注:要返回到以前的某一步复查或修改选项，选择上一步按钮。

  23. 要在此机器上开始 AFS 服务器的配置，选择配置按钮。 AFS 服务器将根据您的说明进行配置。 对话框底部的进程条表明进程的各个步骤。 最后将出现一个消息框，指示配置已完成。

要将 AFS 服务器配置到现有 AFS 单元中：

  1. 在开始菜单，选择程序，然后选择 IBM AFS，再选择服务器，再选择配置向导。 “AFS 服务器快速启动向导”打开。

  2. 选择下一步按钮。 “单元和服务器信息”对话框出现。

  3. 选择使此主机成为现有 AFS 单元中的服务器选项。

  4. 在单元名框中，输入您希望在其中添加新 AFS 服务器的 AFS 单元的名称。

  5. 选择下一步按钮。 “管理信息”对话框出现。

  6. 在名称框中，输入 AFS 管理帐户的用户名，例如 admin，或输入一个有管理特权的 AFS 用户帐户的用户名。

  7. 在口令框中，输入 AFS 管理帐户的口令，或输入在名称框中输入的有管理特权的 AFS 用户帐户的口令。

  8. 在 AFS 服务器框中，输入一个在此 AFS 单元中运行的 AFS 服务器的主机名。 会从服务器中检索 AFS 配置信息，用于配置该新 AFS 服务器。

  11. 选择下一步按钮。 “备份服务器”对话框出现。

    备份服务器维护一个备份数据库，在其中存储与备份系统相关的信息。 备份服务器使 AFS 系统管理员能够将 AFS 文件空间中的数据从卷备份到磁带中。 这样，当文件系统中的数据丢失时（例如，如果由于系统或磁盘故障而引起数据丢失），数据能够从磁带恢复。
    注:备份服务器只能在已配置为数据库服务器的机器上配置。 如果单元中的任何一个数据库服务器配置了备份服务器，则单元中的所有数据库服务器也必须配置。

  12. 如果要将此 AFS 服务器配置为一个备份服务器，选择是，配置为备份服务器选项。 如果您不想将此 AFS 服务器配置为一个备份服务器，选择否，不配置为备份服务器选项。

  13. 选择下一步按钮。 “AFS 分区”对话框出现。

    如果您将此 AFS 服务器配置为一个文件服务器，则必须将某一 NTFS 卷指定为 AFS 分区。 每个 AFS 文件服务器必须至少有一个专门指定为存储 AFS 卷的分区，并且所有的 AFS 卷必须驻留在指定为 AFS 分区的分区上。 在 Windows NT 机器上，只有 NTFS 卷能被指定为 AFS 分区。 另外，AFS 分区只能创建在空的 NTFS 卷上（或仅包含 Windows NT 回收站）。

    要将一个卷指定为 AFS 分区，选择是，创建分区选项。 在 NTFS 卷列表中，选择您要指定为 AFS 分区的卷。 在 AFS 分区名框中，输入分区名的最后部分。
    注:在一个 AFS 服务器上最多可存在 256 个 AFS 分区。 约定情况下，每个分区被命名为 /vicepx， 其中 x 是一个或两个小写英文字母。 AFS 分区可命名为 /vicepa、/vicepb、等等，直到 /vicepz。 其它的分区可命名为 /vicepaa 到 vicepaz，等等，直到 /vicepiv。

    极力推荐您使用 NTFS 卷盘符作为分区名的最后一个字母。

    如果您不想将卷指定为 AFS 分区，选择否，不创建分区选项。

  14. 选择下一步按钮。 “根目录 AFS 卷”对话框出现。

    根目录 AFS 卷是两个卷，每个 AFS 单元必须在其文件系统中包括这两个卷。 它们被命名为：

    • root.afs，用于与 AFS 文件空间的顶层（/afs）相对应的卷

    • root.cell，用于安装在 /afs 目录下单元名称位置的卷（例如，yourcompany.com单元中的 /afs/yourcompany.com）

    因为您将此 AFS 服务器添加到现有 AFS 单元中，所以单元中已存在根目录 AFS 卷，并且 AFS 配置向导指示您无须再创建根目录卷。
    注:如果由于某些原因此 AFS 单元中还没有根目录 AFS 卷，则您可选择是，创建根目录卷选项，在此 AFS 服务器上创建根目录卷。

  15. 选择下一步按钮。 “复制”对话框出现。

    如果您希望利用 AFS 的复制功能，则必须复制 AFS 根目录卷。 复制进程创建 AFS 卷的一个或多个只读副本，并将这些副本分布至一个或多个其它站点（AFS 分区和服务器）。 复制操作使一个 AFS 卷的内容可在一个或多个 AFS 文件服务器机器上存取到，就提高了系统的效率并改进了数据的可用性。

    因为您将此 AFS 服务器添加到现有 AFS 单元中，所以根目录 AFS 卷可能已复制，并且 AFS 服务器配置向导指示您无须再创建根目录 AFS 卷。
    注:如果由于某些原因此 AFS 单元中还没有复制根目录 AFS 卷，则您可选择是，复制根目录卷选项，在此 AFS 服务器上复制 AFS 单元的根目录卷。

  16. 选择下一步按钮。 “系统控制服务”对话框出现。

    在运行美国国内版 AFS Windows 版的单元中，系统控制服务器将新版的 AFS 服务器配置信息分布至所有的 AFS 文件服务器， 并且系统控制客户机机器从系统控制机器中获取公共 AFS 配置文件。 （运行国际版 AFS Windows 版的单元不使用系统控制服务器来分布系统配置文件，或使用系统控制客户机来获取这些文件。）

  17. 要将此 AFS 服务器配置为 AFS 单元的系统控制服务器，选择配置为系统控制服务器选项。 要将此 AFS 服务器配置为一个系统控制客户机，选择配置为系统控制客户机选项，并输入此 AFS 单元中系统控制服务器的主机名。 AFS 服务器将从指定的服务器获取新版的 AFS 服务器配置信息。 如果您不想将此 AFS 服务器配置为 AFS 单元的系统控制服务器或系统控制客户机，选择不配置为系统控制客户机或服务器选项。

  18. 选择下一步按钮。 “配置系统”对话框出现。

    显示配置此 AFS 服务器所要执行步骤的列表，使您可在开始实际的配置进程前复查每一步。
    注:要返回到以前的某一步复查或修改选项，选择上一步按钮。

  19. 要在此机器上开始 AFS 服务器的配置，选择配置按钮。 如果您将 AFS 服务器配置到一个运行老于 AFS 版本 3.5 数据库服务器的 AFS 单元中，则会出现一个对话框，提示您输入 AFS 委托人口令。

    AFS 服务器将根据您的说明进行配置。 对话框底部的进程条表明进程的各个步骤。 最后将出现一个消息框，指示配置已完成。

配置 AFS 控制中心

注:如果您的 AFS 控制中心是与 AFS 服务器和/或 AFS 客户机一起安装的，则不需要配置 AFS 控制中心。 AFS 控制中心在 AFS 服务器或 AFS 客户机配置时已自动配置。 如果您只安装了 AFS 控制中心，则使用前必须在系统上对控制中心进行配置。

  1. 在开始菜单中，选择设置，然后选择控制面板。

  2. 双击 AFS 控制中心图标。 “AFS 控制中心特性”对话框出现。

  3. 在缺省单元框中，输入缺省时要管理单元的全名。

  4. 如果 AFS 控制中心要管理的单元没有列出在 AFS 单元列表中，则选择添加按钮。 “新建单元”对话框打开。 在 AFS 单元框中输入单元名，在描述框中输入一个简短的说明。

    选择添加按钮。 “添加服务器”对话框打开。 在服务器名框中，输入所选单元中某一卷位置服务器的名称。 选择确定以关闭“添加服务器”对话框。 重复此过程，为单元中的所有卷位置服务器添加信息。 在输入了所有服务器的信息后，选择确定以关闭“新建单元”对话框。

  5. 选择确定以关闭“AFS 控制中心特性”对话框。

现在 AFS 控制中心已配置。

卸装 AFS Windows 版

本节略述卸装先决条件，提供卸装 AFS Windows 版的指示信息，并列出卸装进程对系统的更改。

重新安装和升级

在 Windows NT 机器上，无须为重新安装或升级软件而卸装 AFS Windows 版的组件。 要重新安装或升级 AFS Windows 版，请遵循安装 AFS Windows 版中描述的安装步骤。 在安装进程期间，先前安装的 AFS 组件被替换。 AFS 配置信息被保留。

在 Windows 95 或 Windows 98 机器上，重新安装或升级 AFS Light 之前，您必须卸装先前安装的 AFS Light 组件，如卸装 AFS Windows 版中所述。

卸装先决条件

卸装 AFS 将导致所有 AFS 应用程序文件的删除。 如果其它应用程序正在使用某些 AFS 应用程序文件，则这些文件将无法删除。 因此，在您卸装 AFS Windows 版前，必须关闭所有的 AFS 对话框。

如果您是为了停止使用机器而卸装 AFS 服务器，则必须遵循下列先决条件，以防止数据损失：

  1. 如果 AFS 服务器作为文件服务器使用，请将所有读/写卷移到另一 AFS 文件服务器中，并除去所有只读卷。

  2. 解除 AFS 服务器的配置。打开“AFS 服务器配置”实用程序，并选择服务器标签。 清除所有复选框，并选择确定。

卸装 AFS Windows 版

  1. 在开始菜单中，选择设置，然后选择控制面板。

  2. 双击添加/删除程序图标。 “添加/删除程序属性”对话框出现，显示添加/卸载标签。

  3. 关闭控制面板。

  4. 选择要卸装的 AFS 组件，并选择添加/删除按钮。 “Confirm File Deletion”（确认文件删除）对话框出现，验证您要除去选定的 AFS Windows 版组件。 单击是继续卸装过程。

  5. 出现一个 AFS 消息框，询问是否保留配置信息。 选择是保留配置信息，或选择否删除所有配置信息。 （没有与 AFS 补充文档组件关联的配置信息。 因此如果您从组件中除去该组件，将不出现 AFS 消息框。）

  6. “从您的计算机中删除程序”对话框打开，显示从系统中除去的组件。

    注:可能会出现一个消息框，询问您是否要删除不再被其它组件使用的共享 AFS 文件。 单击全是，完全删除选定的 AFS 组件。

选定的 AFS Windows 版组件现在已卸装完毕。 如果您安装有多个 AFS Windows 版组件，则必须重复第 4 步至第 6 步，分别除去每个组件。

对系统的更改

卸装 AFS 客户机对系统的更改

卸装 AFS 客户机对您的系统有下列更改：

  • 删除 \Program Files\Ibm\Afs\Client\Program 目录中所有的 AFS 客户机文件， 删除 Client 目录，并且如果不再安装有其它 AFS 组件，删除 Ibm 目录。
    注:如果目录中包含任何不是由 AFS Windows 版 setup 程序安装的文件，则目录不被删除。

  • 如果不再安装有其它 AFS 组件，则从开始菜单中删除 IBM AFS 程序组。

  • 从控制面板中删除 AFS 客户机配置图标。

  • 从 Windows NT 资源管理器的上下文菜单中删除 AFS 菜单。

  • 删除 AFS Client（AFS 客户机）服务。

  • 从系统中删除与 AFS 客户机相关的注册表条目。 注意，如果您在卸装时选择了保留配置信息，则一些信息会在卸装进程后仍保留在注册表中。

卸装 AFS Light 对系统的更改

卸装 AFS Light 对您的系统有下列更改：

  • 删除 \Program Files\Ibm\Afs\Client\Program 目录中所有的 AFS 文件， 并删除 Ibm 目录。
    注:如果目录中包含任何不是由 AFS Windows 版 setup 程序安装的文件，则目录不被删除。

  • 从开始菜单中删除 IBM AFS 程序组。

  • 从控制面板中删除 AFS Light 配置图标。

  • 从 Windows 资源管理器的上下文菜单中删除 AFS 菜单。

  • 从系统中删除与 AFS Light 相关的注册表条目。 注意，如果您在卸装时选择了保留配置信息，则一些信息会在卸装进程后仍保留在注册表中。

卸装 AFS 服务器对系统的更改

卸装 AFS 服务器对您的系统有下列更改：

  • 删除 \Program Files\Ibm\Afs\Server 目录中所有的 AFS 服务器文件， 删除 Server 目录，并且如果不再安装有其它 AFS 组件，删除 Ibm 目录。
    注:如果目录中包含任何不是由 AFS Windows 版 setup 程序安装的文件，则目录不被删除。 同样，如果您在卸装时选择了保留配置信息，则 \Program Files\Ibm\Afs\Server 目录中的一些文件不被删除。

  • 如果不再安装有其它 AFS 组件，则从开始菜单中删除 IBM AFS 程序组。

  • 从控制面板中删除 AFS 服务器配置图标。

  • 删除 AFS Server（AFS 服务器）服务。

  • 从系统中删除与 AFS 服务器相关的注册表条目。 注意，如果您在卸装时选择了保留配置信息，则一些信息会在卸装进程后仍保留在注册表中。

卸装 AFS 控制中心对系统的更改

卸装 AFS 控制中心对您的系统有下列更改：

  • 删除 \Program Files\Ibm\Afs\Control 目录中所有的 AFS 控制中心文件， 删除 Control Center 目录，并且如果不再安装有其它 AFS 组件，删除 Ibm 目录。
    注:如果目录中包含任何不是由 AFS Windows 版 setup 程序安装的文件，则目录不被删除。

  • 如果不再安装有其它 AFS 组件，则从开始菜单中删除 IBM AFS 程序组。

  • 从控制面板中删除 AFS 控制中心图标。 注意，仅当您的系统上没有安装其它 AFS Windows 版组件时，此图标才会出现在控制面板中。

  • 从系统中删除与 AFS 控制中心相关的注册表条目。 注意，如果您在卸装时选择了保留配置信息，则一些信息会在卸装进程后仍保留在注册表中。

卸装 AFS 补充文档对系统的更改

卸装 AFS 补充文档对您的系统有下列更改：

  • 从 \Program Files\Ibm\Afs\Documentation\Html 目录中删除 SysAdminGd 目录和 CmdRef 目录， 并且如果不再安装有其它 AFS 组件，删除 Ibm 目录。
    注:如果目录中包含任何不是由 AFS Windows 版 setup 程序安装的文件，则目录不被删除。

  • 如果不再安装有其它 AFS 组件，则从开始菜单中删除 IBM AFS 程序组。

  • 从系统中删除与 AFS 补充文档相关的注册表条目。
