AFS 라이트 릴리스 노트

소개

이 절에는 Windows용 AFS 버전 3.6의 AFS 라이트 구성요소에 대한 중요한 정보가 들어 있습니다. AFS 라이트는 Windows 95 또는 Windows 98을 실행하는 PC에서 AFS 파일 공간에 대한 액세스를 제공하며, 사용자가 AFS에 있는 파일과 디렉토리를 관리하게 합니다. AFS 라이트는 AFS 라이트 게이트웨이로서 구성된 AFS 클라이언트 기계를 거쳐 AFS 파일 공간에 액세스합니다.

AFS 라이트 설치 요구조건

AFS 라이트를 설치하려면 사용자 시스템에서 다음과 같은 하드웨어, 소프트웨어 및 관리 특권 요구조건을 충족시켜야 합니다.

하드웨어

AFS 라이트를 설치를 위한 하드웨어 요구조건은 다음과 같습니다.

  • 80486/66 MHz(이상) 마이크로프로세서 또는 펜티엄 기반 마이크로프로세서를 사용하는 PC
  • 최소 16MB의 RAM
  • 최소 4MB의 사용가능한 디스크 공간이 있는 하드 디스크

소프트웨어

AFS 라이트를 설치하는 소프트웨어 요구조건은 다음과 같습니다.

  • Windows Sockets 2 update가 있는 Microsoft Windows 95나 Microsoft Windows 98
  • AFS 라이트 게이트웨이
    주:AFS 라이트 게이트웨이는 AFS 클라이언트, 버전 3.5나 버전 3.6을 실행하는 Windows NT 기계에서 사용 가능해야 합니다. AFS 라이트는 AFS 라이트 게이트웨이를 통해 AFS에 액세스합니다. 세부사항은 AFS 라이트 게이트웨이를 통한 AFS 액세스의 내용을 참조하십시오.

관리 특권 요구조건

AFS 라이트를 설치하고 구성하는 데 필요한 관리 특권은 없습니다.

제품 주의사항

이 절은 Windows용 AFS 버전 3.6의 AFS 라이트 구성요소에 대한 중요하고 일반적인 정보에 초점을 맞추었습니다.

AFS 라이트 게이트웨이를 통한 AFS 액세스

AFS 라이트는 AFS 라이트 게이트웨이를 통해 AFS 파일 공간에 액세스합니다. AFS 라이트를 사용하려면, Windows용 AFS 클라이언트, 버전 3.5나 Windows용 AFS 클라이언트, 버전 3.6을 실행하는 AFS 라이트와 동일한 도메인에 최소한 하나의 기계를 가져야 합니다.

  • AFS 라이트 셀 데이터베이스와 AFS 라이트 게이트웨이 셀 데이터베이스는 동기화되어야 합니다

    AFS 라이트 사용자가 셀에 액세스하려면, 셀에 대한 항목이 AFS 라이트 셀 데이터베이스 (afsdcell.ini 파일)와 AFS 라이트 게이트웨이 셀 데이터베이스 (afsdcell.ini 파일) 양쪽에 존재해야 합니다.

  • AFS 라이트는 AFS 라이트 게이트웨이의 서비스 이름을 분석할 수 있어야 합니다

    AFS 라이트는 게이트웨이 시스템과 통신하기 위해 게이트웨이 기계의 이름을 분석할 수 있어야 합니다. 게이트웨이 시스템 이름은 mach-afs 형식의 게이트웨이 NetBIOS 서비스 이름입니다. 여기서 mach는 최고 11자의 문자로 이루어진 호스트 컴퓨터 이름입니다. 게이트웨이의 NetBIOS 서비스 이름을 클라이언트의 LMHOSTS 파일이나 해당 DNS 또는 WINS 서버에 추가하여 이름 분석을 할 수 있습니다. AFS 라이트 시스템과 AFS 라이트 게이트웨이 시스템이 같은 서브넷에 있는 경우 NetBIOS 브로드캐스트에서 자동으로 이름 분석이 이루어집니다.

  • AFS 라이트 게이트웨이는 Windows 컨텍스트에서 AFS 라이트 사용자를 인증할 수 있어야 합니다

    일단 AFS 라이트 게이트웨이로 구성되면 AFS 클라이언트 시스템은 Windows 컨텍스트에 있는 AFS 라이트 사용자를 인증할 수 있어야 합니다. 이러한 인증은 도메인 사용자 계정 또는 동기화된 시스템 사용자 계정을 통해 이루어집니다. 도메인 사용자 계정은 Windows 도메인의 사용자 계정입니다. 시스템 사용자 계정은 특정 호스트 시스템에서만 유효한 사용자 계정입니다.

    AFS 라이트 게이트웨이가 Windows 도메인으로 구성될 때 AFS 라이트 사용자는 게이트웨이가 속해 있는 도메인의 도메인 사용자 계정 또는 게이트웨이 도메인의 도메인 사용자 계정과 같은 사용자 이름 및 암호를 가진 시스템 사용자 계정으로 로그온해야 합니다.

    시스템 사용자 계정이 이미 사용중인 경우 이 계정은 AFS 라이트 게이트웨이 및 AFS 라이트 시스템에서 동기화해야 합니다. 사용자는 AFS 라이트 게이트웨이 시스템에 정의된 시스템 사용자 계정과 같은 사용자 이름 및 암호를 가진 AFS 라이트 시스템으로 로그온해야 합니다.

    도메인 사용자 계정의 사용은 관리를 단순화하기 위해 권장됩니다.

Windows용 AFS 라이트는 Windows Sockets 2 Update를 요구합니다

AFS 라이트를 Windows 95 기계에서 실행하려면, Windows Sockets 2 update를 아직 설치하지 않은 경우 설치해야 합니다. 이 갱신판은 Microsoft 웹 사이트에서 다운로드 받을 수 있습니다.

AFS 명령행 2진

AFS 라이트가 Windows 98 시스템에서 설치되면, AFS 라이트는 bos, kas, vos 및 pts와 같은 관리 지향 명령 스위트를 포함합니다. Windows 명령 프롬프트에서 이 스위트 내의 명령을 실행하십시오. 명령 집합은 AFS 라이트가 Windows 95 시스템에 설치되는 경우 사용이 불가능합니다.

AFS 라이트는 경로가 ANSI 문자만을 포함하는 디렉토리에 설치되어야 합니다

Windows용 AFS 소프트웨어는 경로에 ANSI 문자만이 들어 있는 디렉토리에 설치해야 합니다(예를 들어, C:\Program Files).

임시 파일

Windows용 AFS 라이트는 경우에 따라 임시 파일을 작성합니다. 그러한 파일이 작성되는 곳을 제어하려면, 선호하는 임시 디렉토리의 전체 경로에 환경 변수(TMP 또는 TEMP)를 설정하십시오. 임시 디렉토리를 지정하지 않는 경우, 임시 파일은 파일을 작성한 프로세스의 현재 작업 디렉토리에 작성됩니다.

AFS 라이트에 대한 자국어 지원

AFS는 양방향 스크립트(Hebrew, Arabic 등)에 대한 지원을 포함하여, AFS 라이트 그래픽 사용자 인터페이스(GUI) 도구 및 문서에 대한 자국어 지원을 제공합니다. 설치된 언어 문자열은 제어판의 국가별 설정 상자 대화 상자에 지정한 대로 사용자 기계의 시스템 기본 로케일에 의해 결정됩니다. 현재 로케일에 대한 언어 문자열이 없는 경우, 영어 문자열이 기본값으로 설치됩니다.
주:기계의 시스템 기본 로케일은 기계에서의 사용자 로케일(사용자가 지정)과 다를 수 있습니다. 그러나, Windows용 AFS가 설치된 언어 문자열은 항상 시스템 기본 로케일에 의해 결정됩니다.

kpwvalid Password Strength Checker는 Windows 시스템에서 사용되지 않습니다

AFS 사용자 암호가 Windows 시스템에서 변경되면, kpwvalid 프로그램은 새 암호를 확인하기 위해 사용되지 않습니다 (UNIX 시스템에서, kpwvalid 프로그램이 kpasswd 프로그램과 동일한 디렉토리에 존재하는 경우, kpwvalid 프로그램은 모든 새 사용자 암호를 확인합니다).

제한사항 및 제약사항

이 절은 Windows용 AFS, 버전 3.6의 AFS 라이트 구성요소에 있는 알려진 제한사항 및 제약사항을 설명합니다.

암호화는 중국어 버전에서 지원되지 않습니다

Microsoft Windows 98의 중국어 버전은 AFS 라이트에서 AFS 라이트 게이트웨이로 전송하는 데 필요한 암호화를 지원하지 않습니다. Microsoft Windows 98의 중국어 버전을 사용할 때 AFS 라이트 사용자가 AFS 토큰을 얻게 하려면, AFS에 있는 암호화가 사용 불가능해야 합니다.

AFS에서 암호화를 사용 불가능하게 하려면, 다음 행을 Windows Autoexec.bat 파일에 추가하십시오. 

set AFS_RPC_ENCRYPT=OFF

AFS 암호는 토큰을 얻을 때 암호화되지 않은 형태로 AFS 클라이언트 게이트웨이로 전송되므로 암호화를 사용 불가능하게 하는 것은 잠재적인 보안 위험을 유발할 수 있음을 기억하십시오.

AFS 파일과 디렉토리로 연결되는 기호 링크

디렉토리에 대한 기호 링크는 디렉토리로서 취급되며, 파일에 대한 기호 링크는 Windows 98 및 Windows 95 시스템에서 파일로서 표시됩니다. 어느 것도 기호 링크로 표시되지 않습니다.

현재, AFS 파일 공간에서 기호 링크를 삭제하거나 작성하기 위해 Windows 인터페이스(예를 들어, Windows 탐색기)를 사용할 수 없습니다. Windows 인터페이스를 사용하여 기호 링크인 디렉토리를 삭제하려고 시도하는 경우, 디렉토리의 내용은 삭제됩니다. 그러나 디렉토리 자체는 삭제되지 않습니다. Windows 인터페이스를 사용하여 기호 링크인 파일을 삭제하려고 시도하는 경우, 목표 파일가 제거되는 것이 아니라 링크가 제거됩니다.

AFS 파일 및 디렉토리에 대한 기호 링크를 작성하고 삭제하려면, Windows용 AFS, 버전 3.6과 함께 제공된 symlink.exe 프로그램을 사용하십시오. Windows NT 명령 프롬프트에서 symlink 명령을 실행하십시오.
