부록 C. afsmonitor 프로그램 통계

이 부록은 afsmonitor 프로그램으로 모을 수 있는 통계를 범주와 섹션별로 그룹짓고 각 필드, 그룹 그리고 섹션을 간단하게 설명하면서 나열합니다. afsmonitor 프로그램 사용 방법은 AFS 성능 모니터 및 감사를 참조하십시오.

캐쉬 관리 프로그램 통계

캐쉬 관리 프로그램 통계 필드는 다음과 같은 섹션과 그룹으로 분류됩니다.

  • PerfStats_section - 성능 통계 섹션.
    • PerfStats_group - 성능 통계 그룹.
    • misc_group - 기타 그룹.
  • Server_UpDown_section - 서버 시작/종료 통계 섹션.
    • FS_upDown_SC_group - 같은 셀 그룹에서의 파일 서버 시작/종료 통계.
    • FS_upDown_OC_group - 다른 셀 그룹에서의 파일 서버 시작/종료 통계.
    • VL_upDown_SC_group - 같은 셀 그룹에서의 VL 서버 시작/종료 통계.
    • VL_upDown_OC_group - 다른 셀 그룹에서의 VL 서버 시작/종료 통계.
  • RPCop_section - RPC 조작 측정 섹션.
    • FS_RPCopTimes_group - 파일 서버 RPC 조작 타이밍 그룹.
    • FS_RPCopErrors_group - 파일 서버 RPC 조작 오류 그룹.
    • FS_RPCopBytes_group - 파일 서버 RPC 전송 타이밍 그룹.
    • CM_RPCopTimes_group - 캐쉬 관리 프로그램 RPC 조작 타이밍.
  • Auth_Access_section - 인증 및 복제 파일 액세스 섹션.
    • Auth_Stats_group - 캐쉬 관리 프로그램 그룹에 대한 인증 정보.
    • Access_Stats_group - 복제되지 않은 파일 액세스 그룹.

이런 섹션과 그룹으로 분류된 모든 캐쉬 관리 프로그램 변수는 아래에 나열되어 있습니다.

성능 통계 섹션(PerfStats_section)

성능 통계 그룹(PerfStats_group)

  • dlocalAccesses: 로컬 셀에서 파일에 대한 데이터 액세스 수.
  • vlocalAccesses: 로컬 셀에서 파일에 대한 stat 액세스 수.
  • dremoteAccesses: 로컬 셀 밖에서 파일에 대한 데이터 액세스 수.
  • vremoteAccesses: 로컬 셀 밖에서 파일에 대한 stat 액세스 수.
  • cacheNumEntries: 캐쉬 항목 수.
  • cacheBlocksTotal: 캐쉬에 대해 구성된 (1K) 블록 수.
  • cacheBlocksInUse: 활발히 사용중인 캐쉬 블록의 수.
  • cacheBlocksOrig: 부팅할 때 캐쉬 블록의 수.
  • cacheMaxDirtyChunks: 허용되는 오염된 캐쉬 청크의 최대 수.
  • cacheCurrDirtyChunks: 오염된 캐쉬 청크의 현재 수.
  • dcacheHits: 로컬 캐쉬에 있는 데이터 파일 수.
  • vcacheHits: 로컬 캐쉬에 있는 stat 항목의 수.
  • dcacheMisses: 로컬 캐쉬에 없는 데이터 파일의 수.
  • vcacheMisses: 로컬 캐쉬에 없는 stat 항목의 수.
  • cacheFlushes: 캐쉬에서 플러쉬된 파일 수.
  • cacheFilesReused: 재사용된 캐쉬 파일의 수.
  • ProtServerAddr: 사용된 보호 서버의 주소.
  • vcacheXAllocs: 추가로 할당된 vcache.
  • bufAlloced: AFS에서 할당된 버퍼 수.
  • bufHits: 버퍼 캐쉬에 있는 페이지 수.
  • bufMisses: 버퍼 캐쉬에 없는 페이지 수.
  • bufFlushDirty: 모두 사용중이므로 플러쉬된 캐쉬 오염 버퍼의 수.
  • LargeBlocksActive: 현재 사용된 사용가능한 큰 풀 항목 수.
  • LargeBlocksAlloced: 할당된 사용가능한 큰 풀 항목 수.
  • SmallBlocksActive: 현재 사용된 사용가능한 작은 풀 항목 수.
  • SmallBlocksAlloced: 할당된 사용 가능한 작은 풀 항목의 수.
  • OutStandingMemUsage: 할당된 메모리의 양.
  • OutStandingAllocs: 미해결 osi_allocs(아직 osi_frees가 없음).
  • CallBackAlloced: 할당된 콜백 구조의 수.
  • CallBackFlushes: 수행된 콜백 플러쉬 조작의 수.
  • srvRecords: 레코드에 현재 있는 서버의 수.
  • srvRecordsHWM: 서버 레코드 고위험 수위.
  • srvNumBuckets: 서버 해시 체인 버켓의 수.
  • srvMaxChainLength: 최대 서버 해시 체인 길이.
  • srvMaxChainLengthHWM: 서버 해시 체인 고위험 수위.
  • sysName_ID: 호스트 하드웨어에 대한 Sysname ID.

기타 그룹(misc_group)

  • numPerfCalls: 수신된 성능 호출의 수.
  • epoch: 캐쉬 관리 프로그램 사건 발생 시간.
  • numCellsVisible: 알고 있는 셀의 수.
  • numCellsContacted: 접속된 셀의 수.
