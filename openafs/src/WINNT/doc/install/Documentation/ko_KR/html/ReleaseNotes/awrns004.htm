AFS 클라이언트 릴리스 노트

소개

이 절에는 Windows용 AFS 버전 3.6의 AFS 클라이언트 구성요소에 대한 중요한 정보가 들어 있습니다. AFS 클라이언트는 Windows NT를 실행하는 PC에서 AFS 파일 공간으로의 직접적인 액세스를 제공하며, 사용자가 AFS에 있는 파일과 디렉토리를 관리하게 합니다. AFS 클라이언트에는 AFS 라이트 사용자가 AFS 파일 공간으로 액세스할 수 있도록 도와주는 AFS 라이트 게이트웨이가 있습니다.

AFS 클라이언트 설치 요구조건

AFS 클라이언트를 설치하기 위해서는 사용자 시스템에 다음과 같은 하드웨어, 소프트웨어 및 관리 특권 요구조건이 충족되어야 합니다.

하드웨어

AFS 클라이언트를 설치하는 데 필요한 하드웨어 요구조건.

  • 80486/66 MHz(이상) 마이크로프로세서 또는 펜티엄 기반 마이크로프로세서를 사용하는 PC
  • 최소 16 MB의 RAM
  • 원하는 캐쉬 크기를 수용하려면 캐쉬가 위치한 드라이브에 최소 4MB의 사용가능한 디스크 공간과 충분한 부가 디스크 공간을 가진 하드 디스크이어야 합니다(기본값으로, 캐쉬는 Windows가 설치되어 있고 캐쉬 크기가 20MB인 드라이브에 위치합니다).

소프트웨어

AFS 클라이언트를 설치하는데 필요한 소프트웨어 요구조건은 다음과 같습니다.

  • 서비스 팩 4 또는 5가 있는 Microsoft Windows NT 4.0

관리 특권 요구조건

AFS 클라이언트를 설치, 구성 및 시작하려면 Windows 시스템에서 국지 관리자 그룹의 구성원이어야 합니다.

제품 주의사항

이 절은 Windows용 AFS 버전 3.6의 AFS 클라이언트 구성요소에 대한 중요하고도 일반 정보에 초점을 맞추었습니다. 여기에는 이 AFS 클라이언트 릴리스에서 사용가능한 새 기능 및 확장사항에 대한 설명이 들어 있습니다.

클라이언트 인터페이스에서 사용가능한 추가 구성 옵션

AFS 클라이언트 구성 유틸리티의 고급 탭에 구성 로그온 매개변수, 진단 매개변수 및 기타 구성 매개변수에 대한 옵션이 들어 있습니다. 고급 탭에서, 전역 드라이브 맵핑을 구성할 수 있습니다. 이전에는 이 구성 옵션 중 많은 것이 등록 설정으로서 AFS 클라이언트 사용자들이 사용할 수 있었습니다.

고급 탭은 Windows 제어판에서 액세스할 수 있습니다. 고급 탭을 표시하려면 다음을 수행하십시오.

  1. 시작 메뉴에서 설정과 제어판을 차례로 선택하십시오.
  2. AFS 클라이언트 구성 아이콘을 두 번 눌러 AFS 클라이언트 구성 유틸리티를 표시합니다.
  3. 고급 탭을 선택하십시오.

이제 AFS 클라이언트 캐쉬 위치를 구성할 수 있습니다

AFS 클라이언트 캐쉬는 시스템 페이징 파일이 아닌 AFSCache 파일에 저장됩니다. 기본값으로, 캐쉬는 Windows가 설치된 드라이브에 저장됩니다. AFS 클라이언트 캐쉬의 크기는 사용가능한 디스크 공간에 의해 제한됩니다. 캐쉬 크기는 최소한 1MB이어야 합니다. 기본 캐쉬 크기는 20480KB(20MB)입니다.

이제 AFS 클라이언트 캐쉬의 위치를 변경할 수 있습니다. 캐쉬 위치를 변경하는 것은 Windows가 설치된 드라이브에 사용가능한 디스크 공간이 충분하지 않을 때 유용합니다. 캐쉬 위치를 변경하려면, AFS 클라이언트 구성의 고급 탭에서 캐쉬 경로 필드에 유효한 완전한 파일 이름을 입력하십시오.

전역 드라이브를 AFS 파일 공간으로 맵핑

AFS 클라이언트의 그래픽 사용자 인터페이스는 이제 AFS 파일 공간에 위치시킬 전역 드라이브를 대응시키는 데 사용될 수 있습니다. IBM AFS 클라이언트 서비스가 시작될 때 전역 드라이브는 AFS 파일 공간으로 맵되며, 사용자는 로그온할 필요가 없습니다. 전역 드라이브를 AFS에 대응시키려면 다음을 수행하십시오.

  1. 시작 메뉴에서 설정과 제어판을 차례로 선택하십시오.
  2. AFS 클라이언트 구성 아이콘을 두 번 눌러 AFS 클라이언트 구성 유틸리티를 표시합니다. 고급 탭을 선택하십시오.
  3. 추가를 누르십시오. 맵 전역 드라이브 대화 상자가 표시됩니다. 드라이브 이름 상자에서, 네트워크 드라이브를 선택하거나 기본값을 채택하십시오. AFS 경로 상자에, AFS 파일 공간에 대한 경로를 입력하십시오. 원하는 경우, 지정된 드라이브 이름과 연관되는 축약형 이름을 입력하고 AFS 경로를 설명 상자에 입력하십시오. 드라이브 이름은 12자 이하여야 하며 공백이나 탭을 포함해서는 안됩니다.
  4. 연결을 설정하려면 확인을 누르십시오. 드라이브 이름 및 AFS 경로가 전역 드라이브 대화 상자에 표시됩니다. 이제 드라이브 이름이 Windows NT 탐색기에 표시됩니다.

이름 서비스 호스트 이름에 해당하는 AFS 클라이언트 컴퓨터 이름

AFS 클라이언트 기계의 컴퓨터 이름(네트워크 대화 상자의 식별 탭에서의 컴퓨터 이름 필드에 표시됨)은 인터넷 프로토콜(IP) 주소로 이름을 맵하기 위해 사용되는 이름 서비스(보통 도메인 이름 서비스 또는 DNS)에 의해 컴퓨터에 지정된 호스트 이름에 대응해야 합니다. 예를 들어, AFS 클라이언트가 설치된 기계가 컴퓨터 이름 afsclient1을 가지고 있으며 yourcompany.com 도메인에 있는 경우, 기계의 해당 DNS 입력항목은 afsclient1.yourcompany.com이어야 합니다.

AFS 명령행 2진

AFS 클라이언트는 bos, kas, vos 및 pts와 같은 관리 지향 명령 세트를 포함합니다. Windows 명령 프롬프트에서 이 세트 내의 명령을 실행하십시오.

AFS 클라이언트는 경로가 ANSI 문자만을 포함하는 디렉토리에 설치되어야 합니다

Windows용 AFS 소프트웨어는 경로에 ANSI 문자만이 들어 있는 디렉토리에 설치되어야 합니다(예를 들어, C:\Program Files).

Windows용 AFS 설정 프로그램의 클라이언트 구성요소를 제외하고 모두 사용 불가능

Windows용 AFS 설정 프로그램을 변경하여 클라이언트 구성요소를 제외한 모든 구성요소를 사용 불가능하게 할 수 있는 옵션이 있습니다. 이러한 클라이언트 전용 설정 프로그램으로는 AFS 클라이언트가 아닌 구성요소는 설치하지 못합니다. 클라이언트 전용 설치를 수행하려면 다른 설치 파일과 같은 디렉토리에 setup.co 파일을 작성하십시오. 그러면 설정 프로그램이 AFS 클라이언트만 설치합니다. setup.co 파일 내용과는 관계 없습니다. 설치 유형에 상관없이 Windows용 IBM AFS 빠른 시작에 설명된 설치 절차를 따르십시오.

임시 파일

Windows용 AFS 클라이언트는 경우에 따라 임시 파일을 작성합니다. 그러한 파일이 작성되는 곳을 제어하려면, 환경 변수(TMP 또는 TEMP)를 선호하는 임시 디렉토리의 전체 경로에 설정하십시오. 임시 디렉토리를 지정하지 않는 경우, 임시 파일은 파일을 작성한 프로세스의 현재 작업 디렉토리에 작성됩니다.

AFS 클라이언트의 자국어 지원

AFS는 양방향 스크립트(Hebrew, Arabic 등)에 대한 지원을 포함하여, AFS 클라이언트 그래픽 사용자 인터페이스(GUI) 도구 및 문서에 대한 자국어 지원을 제공합니다. 설치된 언어 문자열은 제어판의 국가별 설정 상자 대화 상자에 지정한 대로 사용자 기계의 시스템 기본 로케일에 의해 결정됩니다. 현재 로케일에 대한 언어 문자열이 없는 경우, 영어 문자열이 기본값으로 설치됩니다.
주:기계의 시스템 기본 로케일은 기계에서의 사용자 로케일(사용자가 지정)과 다를 수 있습니다. 그러나, Windows용 AFS가 설치된 언어 문자열은 항상 시스템 기본 로케일에 의해 결정됩니다.

kpwvalid Password Strength Checker는 Windows 시스템에서 사용되지 않습니다

AFS 사용자 암호가 Windows 시스템에서 변경되면, kpwvalid 프로그램은 새 암호를 확인하기 위해 사용되지 않습니다 (UNIX 시스템에서, kpwvalid 프로그램이 kpasswd 프로그램과 동일한 디렉토리에 존재하는 경우, kpwvalid 프로그램은 모든 새 사용자 암호를 확인합니다).

구성가능한 LAN(근거리 통신망) 어댑터 번호

AFS 클라이언트 서비스가 사용하는 LAN(근거리 통신망) 어댑터(LANA) 번호는 Windows 시스템에서의 LANA 번호 설정과 일치해야 합니다. 기본값으로, AFS 클라이언트 서비스는 LANA 번호 0을 사용하도록 구성됩니다.

이제 AFS 클라이언트의 그래픽 사용자 인터페이스 도구를 사용하여 AFS 클라이언트 서비스가 사용한 LANA 번호의 값을 수정할 수 있습니다. LANA 번호를 변경하려면, AFS 클라이언트 구성의 고급 탭에서 LAN(근거리 통신망) 어댑터 번호 필드에 새 값을 입력하십시오. 이 매개변수를 수정한 후에 AFS 클라이언트 서비스를 재시작해야 합니다.

대신, Windows NT 기계의 NetBIOS 구성과 일치시키기 위해 AFS 클라이언트 서비스가 사용하는 기본 LANA 번호를 변경하려고 하지 않는 경우, 대신 AFS 기본 설정(0)과 일치시키기 위해 시스템의 LANA 번호 설정을 수정할 수 있습니다. 제어판의 네트워크 응용프로그램에서 서비스 탭에 액세스하십시오. NetBIOS 인터페이스를 선택하고, 등록 정보 단추를 선택하십시오. Lana 번호 필드에 0을 입력하십시오. 이 Windows 설정에 변경사항을 작성한 후에 사용자 기계를 재시작해야 합니다.

WinLogon 그래픽 식별 및 인증 (GINA) 모듈 및 AFS 인증

AFS 클라이언트는 AFS 토큰을 획득하는 WinLogon GINA(Graphical Identification and Authentication) 모듈을 작성하는 데 필요한 헤더 파일 및 라이브러리를 포함합니다. 이 헤더 파일 및 라이브러리는 <Windows용 AFS 설치 디렉토리>\afs\client\program 디렉토리에 설치됩니다. 그리고, 컴파일 및 링크 지시사항이 있는 예제 프로그램이 <Windows용 AFS 설치 디렉토리>\afs\client\program\sample\token.c에서 제공됩니다.

셀 데이터베이스는 갱신 중 유지보수됩니다

AFS 클라이언트 소프트웨어가 업그레이드되면, Windows 디렉토리에 위치한 AFS 클라이언트 셀 데이터베이스 파일(afsdcell.ini)은 국지 셀 구성 정보를 보존하기 위해 바뀌지 않습니다. 그러나, 공급 업체가 분배한 셀 데이터베이스의 사본은 AFS 클라이언트 프로그램 디렉토리에 설치됩니다.

제한사항 및 제약사항

이 절은 Windows용 AFS, 버전 3.6의 AFS 클라이언트 구성요소에서 알려진 제한사항 및 제약사항을 설명합니다.

AFS 파일과 디렉토리로 연결되는 기호 링크

디렉토리에 대한 기호 링크는 디렉토리로서 취급되며, 파일에 대한 기호 링크는 Windows NT 시스템에서 파일로서 표시됩니다. 어느 것도 링크로 표시되지 않습니다.

현재, AFS 파일 공간에 있는 기호 링크를 삭제하거나 작성하기 위해 Windows 인터페이스(예를 들어, Windows NT 탐색기)를 사용할 수 없습니다. Windows 인터페이스를 사용하는 기호 링크인 디렉토리를 삭제하려고 시도하는 경우, 디렉토리의 내용은 삭제됩니다. 디렉토리 자체는 삭제되지 않습니다. Windows 인터페이스를 사용하는 기호 링크인 파일을 삭제하려고 시도하는 경우, 목표 파일가 제거되는 것이 아니라 링크가 제거됩니다.

AFS 파일 및 디렉토리로 연결되는 기호 링크를 작성하고 삭제하려면, Windows용 AFS, 버전 3.6과 함께 제공된 symlink.exe 프로그램을 사용하십시오. Windows NT 명령 프롬프트에서 symlink 명령을 실행하십시오.

AFS 클라이언트는 네트워크 드라이브가 응용프로그램 경로에 있을 때 로드하는 데 실패할 수 있습니다

응용프로그램(AFS 클라이언트와 같은)을 로드할 때, Windows NT는 응용프로그램의 동적링크 라이브러리(DLL)를 탐색합니다. Windows가 응용프로그램의 DLL을 탐색하는 장소 중 하나는 경로 환경 변수에 지정한 각 디렉토리에 있습니다. 경로 환경 변수에서, 네트워크 경로가 DLL을 포함하는 경로 이전에 정의된 경우, 응용프로그램은 로드하는 데 실패할 수 있습니다.

대부분의 서비스(AFS 클라이언트와 같은)는 경로 환경 변수에 지정된 네트워크 드라이브에 대한 보안 액세스를 갖지 않습니다. 네트워크 드라이브가 경로에 있고 응용프로그램이 네트워크를 액세스하는 권한을 갖지 않으면, Windows는 파일에 액세스할 수 없는 프로세스에 의해 야기된 액세스 거부 오류와 파일을 포함하고 있는 네트워크 드라이브에 액세스할 수 없는 프로세스를 구별하지 못합니다. 이러한 구별이 불가능하기 때문에, 운영체제는 파일이 존재하지만 프로세스가 액세스할 수는 없다라고 가정합니다. 이러한 오류는 시스템의 DLL 탐색을 중단하게 하며 DLL이 액세스될 수 없다라고 가정합니다. Windows에서 DDL을 액세스할 수 없으므로, 응용프로그램를 로드하는 데 실패합니다.

이 문제점을 피하기 위해, 응용프로그램의 DLL에 대한 디렉토리가 경로에서 네트워크 드라이브 앞에 오는지 확인하십시오.
