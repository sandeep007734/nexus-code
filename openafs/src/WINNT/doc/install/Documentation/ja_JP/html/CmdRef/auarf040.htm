VolumeItems

目的

ディスク・キャッシュを使用してキャッシュ・マネージャーがアクセスしたボリュームのロケーション・マッピングを記録します。

構文

VolumeItems ファイルには、ディスク・キャッシュを使用するクライアント・マシンで初期化されてからキャッシュ・マネージャーがアクセスしたボリュームごとに、ボリューム名とマウント・ポイントとのマッピングが記録されます。キャッシュ・マネージャーはマッピングを使用して、現在の作業ディレクトリーに関する照会に正しく応答します。この照会は、オペレーティング・システムあるいは UNIX の pwd コマンドなどのコマンドから発行されます。

キャッシュ・マネージャーは、初期化時にローカル・ディスク・キャッシュ・ディレクトリーにバイナリー形式の VolumeItems ファイルを作成します。このファイルは、この位置に常駐している必要があります。標準のディレクトリー名は /usr/vice/cache です。

注意

VolumeItems ファイルを編集したり、削除すると、カーネルに問題が発生する可能性があります。ファイルの内容を更新する場合には、fs checkvolumes コマンドを使用してください。 VolumeItems ファイルが偶発的に変更または削除された場合には、マシンを再始動すると、通常のパフォーマンスに復元されます。
