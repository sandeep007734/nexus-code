目的

fstrace コマンド・セットの概要

構文

fstrace コマンド・セットのコマンドは、システム管理者がデバッグ目的のためにトレース・キャッシュ・マネージャー・オペレーションを使用するインターフェースです。キャッシュ・マネージャー・オペレーションの例は、ファイル・データか、 UNIX ls コマンドのための出力を生成するために使用された状況情報が取り出されます。

fstrace コマンド・インタープリターは、 cm イベント設定 のようなキャッシュ・マネージャー・オペレーションの広範囲な設定を定義します。イベント設定が活動開始された時、キャッシュ・マネージャーは、定義されたオペレーションの 1 つを実行する時のカーネル・メモリーにある cmfx trace log へメッセージを書き込みます。ログは定義されたサイズ(デフォルトでは、60 KB) にまでだけ拡張されます。その後順番に上書きされます(新しい追跡メッセージが古いものを上書きする)。特別な権利のオペレーションが発生する場合、管理者は標準出力にログを後から表示できる、または後の調査のためのファイルへ後から書き込めます。特定な手順の命令の詳細については、AFS 管理の手引き を参照してください。

fstrace コマンド・セットには、いくつかの種類があります。

  • トレース・ログについての情報を管理または表示するコマンド:

    fstrace clear、fstrace lslog、 fstrace setlog

  • イベント設定の状況を設定または表示するコマンド:

    fstrace lsset および fstrace setset

  • トレース・ログの内容を表示するコマンド: fstrace dump
  • ヘルプを呼び出すコマンド: fstrace apropos および fstrace help

オプション

すべての fstrace コマンドは、以下のオプション・フラグを受信します。これは、コマンドの説明でリストされ、ここで詳細に評価されます。

-help
標準出力ストリームにコマンドのオンライン・ヘルプ・メッセージを出力します。このフラグは、コマンドの他のオプションと一緒に使用してはなりません。このフラグを使用すると、コマンド・インタープリターは、他のすべての引き数およびフラグを無視し、ヘルプ・メッセージの出力だけを行います。

必要となる特権

fstrace コマンドのほとんどを発行するには、トレース・ログを生成するマシンにローカル・スーパーユーザー root としてログオンされていなければなりません。
