sysid

目的

VLDB に登録されたファイル・サーバー・マシン・インターフェース・アドレスをリストする。

構文

sysid ファイルには、ファイル・サーバー (fileserver プロセス) がローカル・ファイル・サーバー・マシンをボリューム・ロケーション・データベース (VLDB) に登録するときに使用するネットワーク・インターフェース・アドレスが記録されています。

ファイル・サーバーが再始動するごとに、/usr/afs/local/NetInfo ファイルがある場合には、ファイル・サーバーはこのファイルを読み込み、ローカル・マシン上にインターフェースのリストを作成します。このファイルがない場合には、ファイル・サーバーは、オペレーティング・システムに構成されているネットワーク・インターフェースのリストを使用します。ファイルが存在すると、/usr/afs/local/NetRestrict ファイルにあるすべてのアドレスは、リストから削除されます。ファイル・サーバーは、この結果リストをバイナリー形式の sysid ファイルに記録して、VLDB にそのインターフェースを登録します。

キャッシュ・マネージャーがボリューム・ロケーション情報を要求すると、ボリューム・ロケーション (VL) サーバーは、そのボリュームのあるそれぞれのサーバー・マシンに登録されている、すべてのインターフェースを提供します。これにより、マルチホーム・ファイル・サーバーに保管されている AFS データをアクセスするときに、キャッシュ・マネージャーは複数のアドレスを使用することができます。

注意

sysid ファイルは、各ファイル・サーバー・マシンに固有です。他のマシンにコピーしてはなりません。セルで /usr/afs/local ディレクトリーの内容を既存のファイル・サーバー・マシンから新規にインストールされたマシンにコピーする場合には、新規のマシンから sysid ファイルを削除してから、fileserver プロセスを含む fs トリオのプロセスを開始するのが一般的です。

AFS では、ファイル・サーバー・マシンの、登録できるインターフェース・アドレスの数を制限しているバージョンもあります。 AFS リリース情報 を参照してください。
