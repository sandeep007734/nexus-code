管理解説書

bdb.DB0 および bdb.DBSYS1

目的

バックアップ・データベースおよび関連したログを記録します。

構文

bdb.DB0 ファイルには、バックアップ・データベースが含まれます。このデータベースには、AFS バックアップ・システムが使用する構成情報、バックアップ・システムのコマンドで作成された磁気テープとダンプされたボリュームの相互参照レコードが記録されています。

bdb.DBSYS1 ファイルは、各データベース操作を実行する前にバックアップ・サーバー (buserver プロセス) が各操作を記録したログ・ファイルです。操作が中断すると、バックアップ・サーバーはログを参照して、操作を完了します。

この 2 つのファイルはバイナリー形式で、バックアップ・サーバーを実行しているデータベース・サーバー・マシンの /usr/afs/db ディレクトリーにあります。バックアップ・サーバーが、指定されたマシンで開始または再始動する場合、バックアップ・サーバーは対等なサーバーとの接続を確立し、bdb.DB0 ファイルのこのサーバーのコピーが他のデータベース・サーバー・マシン上のコピーと一致するか検査します。一致していない場合は、バックアップ・サーバーは AFS の分散データベース技術の Ubik を使用して、最新のバージョン番号を持つデータベースのコピーを、すべてのマシンに配布します。

バックアップ・データベースを管理するには、 backup コマンド・セットのコマンドを使用します。 UNIX の tar コマンド、または別のローカル・ディスク・バックアップ・ユーティリティーを使用して、bdb.DB0 ファイルのバックアップ・コピーを一定の間隔で磁気テープに作成するようにしてください。
