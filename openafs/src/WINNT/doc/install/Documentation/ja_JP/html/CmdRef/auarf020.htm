CellServDB (サーバー・バージョン)

目的

ローカル・セルのデータベース サーバー・マシンを表示します。

構文

サーバー・バージョンの CellServDB ファイルには、ローカル・セルのデータベース・サーバー・マシンがリストされます。これらのマシンは、認証サーバー、バックアップ・サーバー、およびボリューム・ロケーション (VL) サーバー (kaserver、buserver、ptserver、および vlserver) プロセスを実行します。これらのプロセスは、セルの管理 AFS データベースを保守します。初期バージョンのファイルは、セルのサーバー・マシンのインストール時に、bos setcellname コマンドで作成されます。このマシンはまた、セルの最初のデータベース・サーバー・マシンとして自動的に記録されます。データベース・サーバー・マシンを追加したり、削除する場合には、必要に応じて、このファイルも更新してください。このファイルは、各 AFS サーバー・マシンの /usr/afs/etc ディレクトリーになければなりません。

データベース・サーバー・プロセスは、CellServDB ファイルを参照して、それらプロセスのピアを認識します。各データベースの複数のコピーに対して変更の複製を調整するために、このピアを使用して一定の接続を維持します。他の AFS サーバー・プロセスは、このファイルを参照して、データベースの情報を参照するマシンを認識します。

CellServDB ファイルは ASCII 形式ですが、テキスト・エディターで更新しないでください。編集する場合には、次のように、bos コマンド・セットから適切なコマンドを使用してください。

  • bos addhost コマンド。ファイルにマシンを追加します。
  • bos listhosts コマンド。ファイルのマシン・リストを表示します。
  • bos removehost コマンド。ファイルからマシンを削除します。

米国版の AFS を実行し、更新サーバーを使用して /usr/afs/etc ディレクトリーの内容を配布しているセルでは、システム制御マシンに保管されているファイルのコピーだけを編集します。国際版の AFS を実行しているセルの場合には、各サーバー・マシンごとにファイルを編集します。データベース・サーバー・マシンの追加および削除に関する説明は、AFS インストールの手引き の、追加するサーバー・マシンのインストールに関する章を参照してください。

サーバー・バージョンの CellServDB ファイルは、各 AFS クライアント・マシン上の /usr/vice/etc ディレクトリーにあるクライアント・バージョンとは異なります。サーバー・バージョンはローカル・セルのデータベース・サーバー・マシンしかリストしませんが、クライアント・バージョンは、セル管理者がマシンのユーザーにアクセスを許可したい AFS セルのデータベース・サーバー・マシンをリストします。
