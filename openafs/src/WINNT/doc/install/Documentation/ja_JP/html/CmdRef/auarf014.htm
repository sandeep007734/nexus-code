BackupLog

目的

バックアップ・サーバーの操作を追跡します。

構文

BackupLog ファイルには、ローカル・マシンでのバックアップ・サーバー (buserver プロセス) 操作と、検出されたエラー状況が記録されます。

バックアップ・サーバーの始動時に BackupLog ファイルが /usr/afs/logs ディレクトリーに存在しない場合は、サーバー・プロセスはファイルを作成し、初期始動メッセージを書き込みます。ファイルが存在する場合は、バックアップ・サーバーはファイル名を BackupLog.old に変更し、既存の BackupLog.old がある場合は、上書きします。

このファイルは、ASCII 形式です。 /usr/afs/etc/UserList ファイルにリストされている管理者は、このファイルの内容を表示する bos getlog コマンドを使用することができます。あるいは、ファイル・サーバー・マシンにログオンして、テキスト・エディターまたは UNIX の cat コマンドなどのファイル表示コマンドを使用します。デフォルトでは、BackupLog ファイル上のモード・ビットは、すべてのユーザーに対して、必須の r (読み取り) アクセス権を許可します。

バックアップ・サーバーは完了した操作のみを記録するので、ファイルを調べるだけでは障害から回復できません。ログの内容はプロセス障害および他の問題の評価を管理するのに有効です。
