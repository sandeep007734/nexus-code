AFS Light 版本注意事項

簡介

本節包含 AFS for Windows 3.6 版之 AFS Light 元件的重要資訊。 AFS Light 可讓執行 Windows 95 或 Windows 98 的 PC 存取 AFS 檔案空間， 讓使用者能夠管理 AFS 中的檔案和目錄。 AFS Light 經由已架構成 AFS Light 閘道的 AFS 用戶端機器來存取 AFS 檔案空間。

AFS Light 安裝基本需求

您的系統必須符合下列硬體、軟體及管理權限基本需求， 才能安裝 AFS Light。

硬體

安裝 AFS Light 的硬體基本需求：

  • 80486/66 MHz（或更快）微處理器或 Pentium 等級微處理器的 PC
  • 至少 16 MB 的 RAM
  • 至少 4 MB 可用磁碟空間的硬碟

軟體

安裝 AFS Light 的軟體基本需求：

  • Microsoft Windows 95 加上 Windows Sockets 2 更新程式，或 Microsoft Windows 98
  • AFS Light 閘道
    註:必須在執行 AFS 用戶端 3.5 版或 3.6 版的 Windows NT 機器上啟用 AFS Light 閘道。 AFS Light 透過 AFS Light 閘道來存取 AFS。若需其它資訊，請參閱經由 AFS Light 閘道存取 AFS。

管理權限基本需求

安裝和架構 AFS Light 不需要管理權限。

產品注意事項

本節強調關於 AFS for Windows 3.6 版之 AFS Light 元件重要的一般資訊。

經由 AFS Light 閘道存取 AFS

AFS Light 經由 AFS Light 閘道存取 AFS 檔案空間。 為了使用 AFS Light，您在相同的領域內，至少必須有一部機器是執行 Windows 版 AFS 用戶端 3.5 版或 Windows AFS 用戶端 3.6 版的 AFS Light 機器。

  • AFS Light cell 資料庫和 AFS Light 閘道 cell 資料庫必須同步化

    為了讓 AFS Light 使用者能夠存取 cell， 該 cell 的登錄必須存在於 AFS Light cell 資料庫（afsdcell.ini 檔案）和 AFS Light 閘道 cell 資料庫（afsdcell.ini 檔案）內。

  • AFS Light 必須能夠轉換 AFS Light 閘道的服務名稱

    AFS Light 必須能夠轉換閘道機器的名稱，以便與閘道機器進行通信。 閘道機器的名稱即為閘道的 NetBIOS 服務名稱，格式為 mach-afs，其中 mach 是主電腦的名稱，最多可達 11 個字元。 您也可以藉由將閘道的 NetBIOS 服務名稱新增到用戶端的 LMHOSTS 檔案，或新增到適當的 DNS 或 WINS 伺服器，來解析名稱。 若 AFS Light 機器及其 AFS Light 閘道機器位於相同的子網路中， 則會經由 NetBIOS 廣播來自動達成名稱轉換。

  • AFS Light 閘道必須能夠鑑別 Windows 環境定義中的 AFS Light 使用者

    一旦架構為「AFS Light 閘道」之後，您的「AFS 用戶端」機器就一定可以鑑別 Windows 中的 AFS Light 使用者。此種鑑別方式可以經由領域使用者帳戶或經由同步的機器使用者帳戶來達成。領域使用者帳戶即為 Windows 領域中的使用者帳戶。 而機器使用者帳戶即為僅在特定主電腦機器上有效的使用者帳戶。

    當 AFS Light 閘道架構在 Windows 領域內之後， AFS Light 使用者必須登入到閘道所屬領域中的領域使用者帳戶， 或以閘道領域中之領域使用者帳戶的相同使用者名稱和密碼，登入機器使用者帳戶。

    如果機器使用者帳戶已經被使用，則這些帳戶必須在「AFS Light 閘道」及 AFS Light 機器上實施同步。 使用者必須使用與「AFS Light 閘道」上所定義之機器使用者帳戶相同的使用者名稱和密碼，來登入 AFS Light 機器。

    建議使用領域使用者帳戶來簡化管理作業。

AFS Light for Windows 95 需要 Windows Sockets 2 更新程式

若要在 Windows 95 機器上執行 AFS Light，您必須先安裝 Windows Sockets 2 更新程式。此更新程式可從 Microsoft Web 網站下載。

AFS 指令行程式

在 Windows 98 系統上安裝時，AFS Light 包含管理專用的指令集，例如 bos、kas、vos、 及 pts。請從「Windows 指令提示」中執行這些指令集。 若在 Windows 95 系統上安裝 AFS Light，則無法使用這些指令集。

必須將 AFS Light 安裝在其路徑只含 ANSI 字元的目錄中

必須將 AFS for Windows 軟體安裝在其路徑只含 ANSI 字元的目錄中， 例如預設目錄 C:\Program Files。

暫存檔

有時候 AFS Light for Windows 會建立暫存檔。若要控制這些檔案的建立位置，請將環境變數（TMP 或 TEMP）設定為您所要之暫時目錄的完整路徑。若不指定暫時目錄， 則將在建立檔案的程序之現行工作目錄中建立任何暫存檔。

AFS Light 的國家語言支援

AFS 提供關於 AFS Light 圖形使用者介面（GUI）工具和說明文件的國家語言支援， 包括雙向 script 的支援（希伯來、阿拉伯等）。安裝的語言字串係由機器的系統預設語言環境所決定， 亦即「控制台」的「國別設定內容」對話方塊中所指定的。若目前的語言環境沒有可用的語言字串，則依預設安裝英語字串。
註:機器的系統預設語言環境可能不同於 使用者語言環境（由使用者指定）。不過，AFS for Windows 所安裝的語言字串，一律由系統預設語言環境所決定。

Windows 系統不使用 kpwvalid 密碼強化檢查器

從 Windows 系統變更 AFS 使用者密碼時，不使用kpwvalid 程式來檢查新密碼的品質。 （在 UNIX 系統上，若 kpwvalid 程式和 kpasswd 程式位於相同目錄中， 則由 kpwvalid 程式檢查新密碼的品質。）

限制

本節簡述 AFS for Windows 3.6 版之 AFS Light 元件的已知限制。

簡體中文版本不支援加密功能

Microsoft Windows 98 的簡體中文版本不支援加密功能， 這項功能是從 AFS Light 傳輸 AFS 密碼到 AFS Light 閘道時所必備的。為了在使用 Microsoft Windows 98 簡體中文版本時， 讓 AFS Light 使用者能夠取得 AFS 記號，必須停用 AFS 中的加密功能。

若要停用 AFS 的加密功能，請在您的 Windows Autoexec.bat 檔案中加入下行： 

set AFS_RPC_ENCRYPT=OFF

請注意：停用加密功能會產生潛在的安全性風險，因為在取得記號時， AFS 密碼是以未加密的格式傳輸到 AFS 用戶端閘道。

AFS 檔案和目錄的符號鏈結

目錄的符號鏈結被視為目錄，檔案的符號鏈結在 Windows 98 和 Windows 95 系統中是以檔案形式出現。兩者皆不以鏈結符號的形式呈現。

目前，您無法使用 Windows 介面（例如 Windows 檔案總管）來刪除或建立 AFS 檔案空間中的符號鏈結。若您嘗試使用 Windows 介面來刪除符號鏈結的目錄， 則會刪除目錄的內容。目錄本身不會被刪除。 若您嘗試使用 Windows 介面來刪除符號鏈結的檔案， 則會移除鏈結，而非刪除目標檔案。

若要建立或刪除 AFS 檔案和目錄的符號鏈結，請使用 AFS for Windows 3.6 版 提供的 symlink.exe 程式。請從 Windows NT 指令提示中執行 symlink 指令。
