IBM AFS for Windows 快速入門

簡介

AFS(R) 是一種透過共用檔案空間的方式，提供一致性檔案存取的企業檔案系統。AFS 透過結合數個「檔案伺服器」機器的本端檔案系統，呈現出一種與機器界限無關的單一檔案空間。如此，檔案雖儲存在電腦網路中不同的機器上，但卻可以從整個企業內的所有機器上存取這些檔案。

IBM AFS for Windows(R) 3.6 版將 AFS 的功能延伸到 Microsoft(R) Windows 作業系統。

文件概觀

本文件彙總了安裝的必備需求，提供安裝、架構，及解除安裝 AFS for Windows 的詳細指示，並指出在安裝和解除安裝的程序期間對系統所做的變更。本文件也說明了 AFS for Windows 所提供的文件。

適用對象

本文件所提供的資訊適用於安裝和架構 AFS for Windows 內之產品的系統管理者和使用者。文中假設系統管理者熟悉一般的系統管理，而使用者熟悉 Microsoft Windows 作業系統的基本術語和概念。

  • AFS for Windows 文件列出 AFS for Windows 所提供的幾種文件類型，並討論使用這些文件的程序。

安裝 AFS for Windows

本節指出 AFS 元件可安裝的組合，說明安裝 AFS for Windows 內所包含之產品的程序，並列出安裝程序對您的系統所做的變更。

AFS for Windows 的元件

AFS for Windows 3.6 版包含了下列元件：

  • AFS 伺服器

    「AFS 伺服器」負責執行 AFS 伺服器程序，它也包含了可協助您進行安裝的「AFS 伺服器架構精靈」。

  • AFS 控制中心

    「AFS 控制中心」包含了兩個功能強大的圖形式使用者介面 (GUI) 工具， 以協助 AFS 系統管理者管理 AFS cell：「AFS 伺服器管理程式」和「AFS 帳戶管理程式」。

  • AFS 用戶端

    「AFS 用戶端」提供對 AFS 檔案空間的直接存取，讓使用者可管理 AFS 中的檔案和目錄。「AFS 用戶端」中包含了「AFS Light 閘道」。

  • AFS Light

    AFS Light 透過一個「AFS Light 閘道」機器來存取 AFS 檔案空間，讓使用者可管理 AFS 中的檔案和目錄。

  • AFS 補充文件

    「AFS 補充文件」提供了附加的 AFS 系統管理資訊，並包含了下列文件：IBM AFS Administration Guide 和 IBM AFS Administration Reference。

安裝選項

您可以根據您的 Windows 作業系統，以各種組合來安裝 AFS for Windows 的元件。請參閱 IBM AFS for Windows 版本注意事項，以獲悉每一個 AFS for Windows 元件對特定軟體之需求明細。請注意：除非機器上已經安裝 AFS 用戶端，否則，如果您是安裝「AFS 伺服器」，則您必須也要在機器上安裝「AFS 用戶端」3.6 版。無論要安裝哪些元件，請遵循安裝 AFS for Windows中說明的安裝程序進行。
註: 您可以選擇將 AFS for Windows 安裝程式改變成停用用戶端元件以外的所有元件。此種只有用戶端的安裝程式使得使用者無法安裝「 AFS 用戶端」以外的任何元件。若要執行只有用戶端的安裝，請在安裝檔案的同一個目錄下建立一個 setup.co 檔案；然後安裝程式便只會允許安裝「AFS 用戶端」。請注意：這和 setup.co 檔案的內容沒有關係。無論您是執行何種安裝，請遵循安裝 AFS for Windows中說明的安裝程序進行。

從早期的版本升級

在 Windows NT 機器上，升級軟體之前並不需要解除安裝 AFS for Windows 的元件；您可以直接在系統上安裝此版次的 AFS for Windows，而不必移除或取消架構現存的軟體。若要升級 AFS for Windows，請遵循 安裝 AFS for Windows 中說明的安裝程序進行。 在安裝程序期間，早期版本的 AFS 元件會被升級，而 AFS 架構資訊會被保留。

在 Windows 95 或 Windows 98 的機器上，您必須先解除安裝之前所安裝的 AFS Light 元件（如解除安裝 AFS for Windows中的說明），才能升級 AFS Light。

請注意：AFS for Windows 安裝工具不允許使用者安裝版本號碼不同的 AFS 元件。如果您的機器上安裝有一個以上的 AFS for Windows 元件，必須要一併更新所有其它的元件，因為無法讓您只更新其中一個元件。

安裝 AFS for Windows

在安裝 AFS for Windows 之前，請先參考 IBM AFS for Windows 版本注意事項，以獲悉安裝必備需求的詳細說明。如果您正在執行其它的 Windows 應用程式，建議您先結束這些應用程式，然後再安裝 AFS for Windows。

  1. 將 AFS for Windows 安裝光碟插入光碟機。
  2. 使用下列其中一種方法來執行 AFS for Windows 安裝程式：
    • 從開始功能表中，選取執行。鍵入 drive:\setup， 其中 drive 是光碟機的磁碟機代號。 按一下確定。
    • 在 Windows 檔案總管中，選取您的光碟機，並按兩下 setup.exe 程式。
  3. 此時「歡迎使用」的對話方塊會出現。請選取下一步按鈕，以繼續安裝程序。
  4. 「選取元件」對話方塊會出現。
    • 在元件方塊中，選取您要安裝或升級的 AFS for Windows 元件。請參閱 安裝選項，以獲悉可在 Windows 機器上安裝之各種元件組合的相關資訊。請注意：如果您是安裝「AFS 伺服器」，您必須也要在系統上安裝（或升級）「AFS 用戶端」3.6 版。
    • 目的資料夾方塊指出所選取之元件將安裝的預設磁碟機和目錄。預設磁碟機即為安裝 Windows 的磁碟機。預設的目錄為 \Program Files\Ibm\Afs。若要選取其它的磁碟機或目錄，請選擇瀏覽按鈕。
      註:如果您是從早期版本的 AFS for Windows 升級，或是重新安裝 AFS for Windows，則您所選擇的安裝目錄必須和之前安裝版本的目錄相同。

    選取下一步按鈕繼續安裝程序。

  5. 然後所選取的 AFS for Windows 元件的應用程式檔案便會安裝到您的系統上。當安裝程序完成後，會出現「安裝完成」對話方塊，指出您必須先重新啟動系統，之後才能使用所安裝的 AFS 產品。請選擇是，我要現在重新啟動電腦，然後選擇完成按鈕。您的系統會關機，然後重新啟動。

    如此 AFS for Windows 的安裝便告完成。

對系統所做的變更

本節說明安裝每一個 AFS for Windows 元件對系統所做的變更。本節的資訊乃根據預設安裝的設定而來。

安裝 AFS 用戶端對系統所做的變更

在 Windows NT 上安裝「AFS 用戶端」會對系統做以下的變更：

  • 建立一個名為 IBM AFS 的開始功能表程式群組，其中含有下列應用程式：

    文件程式項目可讓您存取 AFS for Windows 所提供的 AFS 線上文件集。

    用戶端程式子群組可讓使用者存取「AFS 用戶端」內容方塊及「AFS 用戶端線上說明」。

  • 將「AFS 功能表」加入「Windows NT 檔案總管」的上下文功能表。
  • 建立一個文件目錄，並將 IBM AFS for Windows 快速入門及 IBM AFS for Windows 版本注意事項線上文件放到這個目錄中，該目錄位於 \Program Files\Ibm\Afs\Documentation。
  • 新增 AFS 認證到啟動程式群組。AFS 用戶端圖示會在啟動時顯示在工作列上。
  • 建立安裝目錄，安裝程式會將 AFS 程式檔、圖示，及說明檔安裝到這個目錄中。預設的目錄為 \Program Files\Ibm\Afs\Client\Program 及 \Program Files\Ibm\Afs\Common。
  • 將「AFS 用戶端」登記為服務程式。
  • 安裝「AFS 用戶端架構」公用程式並新增 AFS 用戶端 架構圖示到「控制台」中（藉由將 afs_cpa.cpl 檔案放到 \WindowsDefault\system32 目錄下，其中 WindowsDefault 即為您的 Windows 目錄）。
  • 將 afsdcell.ini 檔案放置在您的 Windows 目錄及 \Program Files\Ibm\Afs\Common 目錄下。 如果您已經從先前的「AFS 用戶端」升級，則 Windows 目錄下的「AFS 用戶端」cell 資料庫 (afsdcell.ini)便不會被取代。
  • 修改「Windows NT 系統登錄」，加入與「AFS 用戶端」有關的項目。

安裝 AFS Light 對系統所做的變更

在 Windows 95 及 Windows 98 上安裝 AFS Light 會對系統做下列的變更：

  • 建立一個名為 IBM AFS 的程式群組，其中含有下列應用程式：

    文件程式項目可讓您存取 AFS for Windows 所提供的 AFS 線上文件集。

    Light 程式子群組可讓使用者存取「AFS Light」內容方塊及「AFS Light 線上說明」。

  • 將「AFS 功能表」加入「Windows 檔案總管」的上下文功能表。
  • 建立一個文件目錄，並將 IBM AFS for Windows 快速入門及 IBM AFS for Windows 版本注意事項線上文件放到這個目錄中，該目錄位於 \Program Files\Ibm\Afs\Documentation。
  • 建立安裝目錄，安裝程式會將 AFS 程式檔、圖示，及說明檔安裝到這個目錄中。預設的目錄為 \Program Files\Ibm\Afs\Client\Program 及 \Program Files\Ibm\Afs\Common。
  • 安裝「AFS Light 架構」公用程式並新增 AFS Light 架構圖示到「控制台」中（藉由將 afs_cpa.cpl 檔案放到 \WindowsDefault\system 目錄下，其中 WindowsDefault 即為您的 Windows 目錄）。
  • 將 afsdcell.ini 檔案放置在您的 Windows 目錄及 \Program Files\Ibm\Afs\Common 目錄下。
  • 修改「Windows 登錄」，加入與「AFS Light」有關的項目。

  • 建立一個名為 IBM AFS 的開始功能表程式群組，其中含有一個名為文件的程式項目：

AFS for Windows 文件

本節說明 AFS for Windows 所提供的文件，並詳述存取此文件的程序。

線上文件目錄

無論您在系統上安裝何種元件，都會建立一個文件目錄。預設的位置是在 \Program Files\Ibm\Afs\Documentation。這個目錄包含了 IBM AFS for Windows 快速入門及 IBM AFS for Windows 版本注意事項。這些相同的文件都可以從開始功能表中的文件項目的「文件」索引存取。

如果您有安裝「AFS 補充文件」，那麼文件目錄也會包含下列文件：IBM AFS Administration Guide及 IBM AFS Administration Reference。這些相同的文件都可以從開始功能表中的文件項目的「文件」索引存取。

存取線上文件目錄：

  1. 從開始功能表中，選擇程式集，選擇 IBM AFS，然後選擇文件。
  2. 選擇您要檢視的文件。

光碟文件目錄

AFS for Windows 的光碟也包含了一個文件目錄。這個目錄包含了下列文件：IBM AFS for Windows 快速入門、IBM AFS for Windows 版本注意事項、IBM AFS Administration Guide，及 IBM AFS Administration Reference。這些文件都是 HTML 和 PDF 格式。

存取光碟文件目錄：

  1. 將 AFS for Windows 光碟插入電腦的光碟機中。
  2. 遵循下列的其中一個路徑。請注意：CD 是光碟機的磁碟機代號。
    • 若要檢視 HTML 文件，請到路徑 CD:\Documentation\Html。
    • 若要檢視 PDF 文件，請到路徑 CD:\Documentation\Pdf。

線上說明

線上說明是隨著每一個 AFS for Windows 元件而安裝的。線上說明文件說明了每一個元件可用的特性。 您可使用大多數對話方塊上的說明功能表和說明按鈕來存取線上說明。您可以瀏覽內容頁以取得某個主題的說明、使用索引來找出主題，並使用線上說明搜尋引擎的尋找功能。

架構 AFS for Windows

本節詳述每一個 AFS for Windows 元件的架構程序。您必須先架構系統上的元件，才能使用 AFS。

架構 AFS 用戶端

註:如果您打算在 Windows NT 系統上架構「AFS 伺服器」，就不需要架構「AFS 用戶端」。當「AFS 伺服器」架構好之後，會自動地架構「AFS 用戶端」。此外，如果您是從先前的「AFS 用戶端」升級到新版的 AFS for Windows，則先前的架構資訊都會被保留。如此也不需要重新架構「AFS 用戶端」。
  1. 從開始功能表中，選擇設定，然後選擇控制台。
  2. 按兩下 AFS 用戶端架構圖示。 「AFS 用戶端架構」公用程式會開啟，並顯示出一般事項標籤。
  3. 在 Cell 名稱方塊中，輸入要成為用戶端之機器所在的 AFS cell 名稱。
  4. 選取 AFS Cell 標籤。如果要成為用戶端之機器所在的 cell 沒有出現在 AFS cell 清單中，請選擇新增按鈕。「新增 Cell」對話方塊會開啟。在 AFS Cell 方塊中輸入 cell 名稱，並在說明方塊中輸入簡短的說明。

    選擇新增按鈕。「新增伺服器」對話方塊會開啟。在伺服器名稱方塊中，於所選取的 cell 內輸入「容體位置伺服器」的名稱。選擇確定關閉「新增伺服器」對話方塊。 重複此程序，加入 cell 中所有「容體位置伺服器」的資訊。（如果您不知道 AFS cell 中的「容體位置伺服器」名稱，請洽詢您的系統管理者。）在輸入完所有的伺服器資訊之後，選擇確定關閉「新增 cell」對話方塊。

  5. 選取一般事項標籤，並選擇啟動服務按鈕來啟動「AFS 用戶端」服務。
  6. 選取磁碟機代號標籤。若要將 Windows NT 機器上的磁碟機代號對應至 AFS 檔案空間，請選擇新增按鈕。「對應磁碟機代號」對話方塊會開啟。
  7. 在磁碟機代號方塊中，選取要對應到 AFS 檔案空間的磁碟機，或接受預設值。在 AFS 路徑方塊中，指出要對應所選取之磁碟機的 AFS 位置，例如， /afs。 如果想要的話，可以在說明方塊中輸入 AFS 磁碟機對應的說明。選擇確定即可將磁碟機連接到 AFS 檔案空間中指定的位置。
  8. 選擇確定關閉「AFS 用戶端架構」公用程式。

    「AFS 用戶端」現在已經架構在所選取的 AFS cell 中，並且也可以透過「Windows NT 檔案總管」中所選取的磁碟機對應來存取 AFS 檔案空間。

將 AFS 用戶端架構為 AFS Light 閘道

您可以將 Windows NT 機器上的「AFS 用戶端」架構為「AFS Light 閘道」。將「AFS 用戶端」架構為「AFS Light 閘道」，便可以讓 AFS Light 的使用者存取 AFS 檔案空間。

  1. 架構「AFS 用戶端」的步驟詳述於架構 AFS 用戶端 中。
  2. 從開始功能表中，選擇設定，然後選擇控制台。
  3. 按兩下 AFS 用戶端架構圖示。 「AFS 用戶端架構」公用程式會開啟，並顯示出一般事項標籤。
  4. 選取提供 AFS Light 閘道選項。
  5. 選擇確定。
    • 如果「AFS 用戶端」服務正在執行中，便會出現一個訊息框，通知您必須要重新啟動服務。選擇是以重新啟動「AFS 用戶端」服務並啟用「AFS Light 閘道」。
    • 如果「AFS 用戶端」是停止的狀態，便會出現一個訊息框，通知您必須要啟動「AFS 用戶端」服務。選擇是以啟動「AFS 用戶端」服務並啟用「AFS Light 閘道」。
  6. 新增 cell 項目到「AFS Light 閘道」的 cell 資料庫。請注意：為了要讓 AFS Light 的使用者能夠存取 cell， 在「AFS Light」cell 資料庫及「AFS Light 閘道」中都必須存在有該 cell 的項目。若閘道機器的 cell 資料庫中和 cell 有關的資訊不正確或遺失，會導致 Light client 的機器無法存取檔案。

    新增項目到 cell 資料庫中：

    從「AFS Light 架構」公用程式進入 AFS Cell 標籤，並選擇新增按鈕。「新增 Cell」對話方塊會開啟。 在 AFS Cell 方塊中輸入 cell 名稱，並在說明方塊中輸入簡短的說明。

    選擇新增按鈕。「新增伺服器」對話方塊會開啟。在伺服器名稱方塊中，自所選取的 cell 內輸入「容體位置伺服器」的名稱。 選擇確定關閉「新增伺服器」對話方塊。重複此程序，加入 cell 中所有「容體位置伺服器」的資訊。（如果您不知道 AFS cell 中的「容體位置伺服器」名稱，請洽詢您的系統管理者。）在輸入完所有的伺服器資訊之後，選擇確定關閉「新增 Cell」對話方塊。

Windows NT 機器現在已經架構為「AFS Light 閘道」了。一旦架構為「AFS Light 閘道」之後，您的「AFS 用戶端」機器就一定可以鑑別 Windows 中的 AFS Light 使用者。此種鑑別方式可以經由領域使用者帳戶或經由同步的機器使用者帳戶來達成。領域使用者帳戶即為 Windows 領域中的使用者帳戶。 而機器使用者帳戶即為僅在特定主電腦機器上有效的使用者帳戶。

當「AFS Light 閘道」架構到 Windows 領域之後，AFS Light 使用者必須登入到閘道所屬之領域中的領域使用者帳戶，或者使用與閘道領域中之領域使用者帳戶相同的使用者名稱和密碼登入到機器使用者帳戶。

如果機器使用者帳戶已經被使用，則這些帳戶必須在「AFS Light 閘道」及 AFS Light 機器上實施同步。使用者必須使用與「AFS Light 閘道」上所定義之機器使用者帳戶相同的使用者名稱和密碼，來登入 AFS Light 機器。

架構 AFS Light

AFS Light 經由「AFS Light 閘道」來存取 AFS 檔案空間。在架構 AFS Light 之前，您必須使 Windows NT 機器執行「AFS 用戶端」，並將其架構為「AFS Light 閘道」。若需其它資訊，請參閱將 AFS 用戶端架構為 AFS Light 閘道。

  1. 從開始功能表中，選擇設定，然後選擇控制台。
  2. 按兩下 AFS Light 架構圖示。「AFS 用戶端 架構」公用程式會開啟，並顯示一般事項標籤。
  3. 在閘道方塊中，輸入架構為「AFS Light 閘道」之 Windows NT 機器的名稱，並按一下立即連接。閘道機器的名稱即為閘道的 NetBIOS 服務名稱，格式為 mach-afs，其中 mach 是主電腦的名稱，最多可達 11 個字元。AFS Light 必須要能夠解析這個服務名稱，才能與閘道機器通信。您也可以藉由將閘道的 NetBIOS 服務名稱新增到用戶端的 LMHOSTS 檔案，或新增到適當的 DNS 或 WINS 伺服器，來解析名稱。如果 AFS Light 機器和它的「AFS Light 閘道」機器是在同一個次網路上，那麼就不需要執行額外的設定。

    AFS Light 會自動地成為相同 cell 的成員，如同它的「AFS Light 閘道」一樣。cell 的名稱會顯示在 Cell 名稱方塊中。
    註:如果「AFS Light 閘道」機器位於和 AFS Light 機器相同的領域中，且此領域中的閘道機器主電腦名稱為 xyz-pc，您便可以在閘道方塊中，指定電腦名稱為 xyz-pc 或 xyz-pc.xcompany.com。

  4. 選取 AFS Cell 標籤。如果機器所在的 cell 沒有出現在 AFS cell 清單中，請選擇新增按鈕。 「新增 Cell」對話方塊會開啟。在 AFS Cell 方塊中輸入 cell 名稱，並在說明方塊中輸入簡短的說明。

    選擇新增按鈕。「新增伺服器」對話方塊會開啟。 在伺服器名稱方塊中，在所選取的 cell 內輸入「容體位置伺服器」的名稱。 選擇確定關閉「新增伺服器」對話方塊。重複此程序，加入 cell 中所有「容體位置伺服器」的資訊。（如果您不知道 AFS cell 中的「容體位置伺服器」名稱，請洽詢您的系統管理者。） 在輸入完所有的伺服器資訊之後，選擇確定關閉「新增 Cell」對話方塊。

    請注意：「AFS Light 閘道」的 cell 資料庫（afsdcell.ini 檔案）必須存在相同的項目，才能使 AFS Light 使用者可鑑別 cell。若需其它有關將閘道機器的 cell 資料庫與 Light 用戶端的 cell 資料庫同步的資訊，請參閱將 AFS 用戶端架構為 AFS Light 閘道。

  5. 選取磁碟機代號標籤。若要將 Windows 機器上的磁碟機代號對應至 AFS 檔案空間，請選擇新增按鈕。 「對應磁碟機代號」對話方塊會開啟。
  6. 在磁碟機代號方塊中，選取要對應到 AFS 檔案空間的磁碟機，或接受預設值。在 AFS 路徑方塊中，指出要對應所選取之磁碟機的 AFS 位置，例如， /afs。 如果想要的話，可以在說明方塊中輸入 AFS 磁碟機對應的說明。選擇確定即可將磁碟機連接到 AFS 檔案空間中指定的位置。
  7. 選擇確定關閉「AFS Light 架構」公用程式。

    「AFS 用戶端」現在已經架構在指定的 AFS cell 中，並且也可以透過「Windows 檔案總管」中的磁碟機對應來存取 AFS 檔案空間。

架構 AFS 伺服器

架構程序會啟動所需的服務，以執行「AFS 伺服器」，並設定 Windows NT 機器上的 AFS 分割區。 使用「AFS 架構精靈」，您可以快速地將「AFS 伺服器」架構為現有的 AFS cell 中的一個伺服器，或架構為新的 AFS cell 中的第一個伺服器。 請注意：如果此版本的 AFS 伺服器 是由升級而來的，則前一版的架構資訊會被保留；您就不需要重新架構伺服器了。

將 AFS 伺服器架構為 cell 中的第一個 AFS 伺服器：

  1. 從開始功能表中，選擇程式集，選擇 IBM AFS，然後選擇伺服器，接著選擇架構精靈。「AFS 伺服器快速啟動精靈」會開啟。
  2. 選擇下一步按鈕。「Cell 和伺服器資訊」對話方塊會出現。
  3. 選擇「這將會是新 AFS cell 中的第一部伺服器」選項。
  4. 在「Cell 名稱」方塊中，輸入新 AFS cell 的名稱。

    下列的限制適用於可作為 AFS cell 名稱之網際網路領域名稱的型式：

    • 這個 cell 名稱必須是獨特的，才能使 AFS cell 與 AFS 整體名稱空間中的其它 cell 有所區別。
    • cell 名稱最多可包含 64 個字元；但是，建議您使用較短的名稱。
    • cell 名稱必須只能包含小寫的字元、數字、底線，及句點，以確保異質作業系統類型之間的可攜性。
    • cell 名稱可包含任意數字或字母，依慣例是以句點來分隔。
    • cell 名稱必須以指示其所屬之機構類型的字尾結束。 標準的字尾有 .com，其代表企業或其它商業組織；.edu 代表教育機構，例如大學；.gov 代表政府機構；而 .mil 代表軍事機構。
  5. 在密碼方塊中輸入字串，作為 cell (afs)中「AFS 伺服器」 principal 帳戶的密碼。所有的「AFS 伺服器」會取得 AFS 記號作為此 principal，且「鑑別伺服器」的「通行證認可服務」(TGS) 模組會使用此密碼，將共同鑑別的「AFS 用戶端」呈現給伺服器的伺服器通行證予以加密。
  6. 在驗證密碼方塊中，再次輸入此 cell 之「AFS 伺服器」principal 帳戶的起始 AFS 密碼，以確認所選擇的密碼。
  7. 選擇下一步按鈕。「管理資訊」對話方塊就會出現。
  8. 在名稱對話方塊中，輸入一個作為此 cell 之通用 AFS 管理帳戶的使用者名稱（通常是 admin）。

    使用通用的管理帳戶表示您並不需要授與專用權給每一個系統管理員。相反地，每一個管理員都知道此通用帳戶的名稱和密碼，並在執行需要管理權限的作業時，使用此身份識別來鑑別 AFS。

  9. 在密碼方塊中，輸入字串，作為 AFS 管理帳戶的密碼。
  10. 在驗證密碼方塊中，再次輸入 AFS 管理帳戶的密碼，以確認所選擇的密碼。
  11. 設定「AFS 使用者 ID」(UID) 以指定給 AFS 管理帳戶：
    • （建議）若要自動地指定下一個可用的 UID 給 AFS 管理帳戶，請選擇使用下一個可用的 AFS UID 選項。
    • 若要指定一個特定的 UID 給 AFS 管理帳戶，請選擇使用此 AFS ID 選項，並在輸入框中輸入想要的 UID。
      註:一般建議您不要指定特定的 UID 給新的 AFS 帳戶，除非您想要使 AFS UID 符合現有的 UNIX UID。
  12. 選擇下一步按鈕。「檔案服務」對話方塊就會出現。

    「AFS 檔案伺服器」會將所要求的檔案和資料從伺服器傳遞到「AFS 用戶端」。 「檔案伺服器」會儲存檔案和資料，處理複製、移動、建立，及刪除檔案和目錄的要求；追蹤伺服器上每一個檔案和目錄的狀態資訊。

    由於您是在新的 cell 中架構第一個「AFS 伺服器」，因此必須在伺服器上架構「檔案服務」，「檔案服務」會自動地被架構。

  13. 選擇下一步按鈕。「資料庫服務」對話方塊就會出現。

    每一個 AFS cell 都必須至少包含一個「資料庫伺服器」。 每一個「資料庫伺服器」都執行了維護 AFS 資料庫的資料庫程序：「鑑別資料庫」、「保護資料庫」 、「容體位置資料庫」，及選擇性的「備份資料庫」。

    由於您是在新的 cell 中架構第一個「AFS 伺服器」，因此必須在伺服器上架構「資料庫服務」，「資料庫服務」會自動地被架構。

  14. 選擇下一步按鈕。「備份伺服器」對話方塊就會出現。

    「備份伺服器」負責維護「備份資料庫」，其中儲存了與「備份」系統相關的資訊。 「備份伺服器」使得 AFS 系統管理員可以將 AFS 檔案空間中的資料從容體備份到磁帶中。 當發生資料從檔案系統中遺失的狀況時（例如，假設系統或磁碟故障而導致資料遺失），便可以從磁帶復置。

  15. 若要將此「AFS 伺服器」架構為「備份伺服器」，請選擇是，架構為備份伺服器選項。如果您不要將此「AFS 伺服器」架構為「備份伺服器」，請選擇「否，不要架構為備份伺服器」選項。

    註:而且，如果「備份伺服器」被架構在 cell 中的任何「資料庫伺服器」上，則它也必須架構在 cell 中的所有「資料庫伺服器」上。

  16. 選擇下一步按鈕。「AFS 分割區」對話方塊就會出現。

    每一部「AFS 檔案伺服器」至少都必須有一個專門用來儲存 AFS 容體的分割區，而所有的 AFS 容體都必須常駐在已經被指定為 AFS 分割區的分割區上。 在 Windows NT 的機器上，只有 NTFS 容體可被指定為 AFS 分割區。 除此之外，AFS 分割區只能建立在空的（或只包含「Windows NT 資源回收筒」的磁碟分割區）NTFS 容體上。

    由於您是在新的 cell 中架構第一個「AFS 伺服器」，因此您必須在伺服器上指定一個 AFS 分割區。

  17. 在 NTFS 容體清單中，選擇您要指定為 AFS 分割區的分割區。 在 AFS 分割區名稱方塊中，輸入分割區名稱的最後一個部分。
    註:一部「AFS 伺服器」上最多可存在有 256 個 AFS 分割區。 依照慣例，每個分割區都命名為 /vicepx，其中 x 是一個或兩個英文字母的小寫字母。AFS 分割區可命名為 /vicepa、/vicepb，依此類推一直到 /vicepz。其它的分割區可命名為 /vicepaa 到 vicepaz，依此類推一直到 /vicepiv。

    強烈建議您使用 NTFS 容體的磁碟機代號作為分割區名稱的最後一個部分。

  18. 選擇下一步按鈕。「AFS Root 容體」對話方塊就會出現。

    AFS 根容體是兩個容體，每一個 AFS cell 都必須包含在其檔案系統中。 它們分別為：

    • root.afs，它是對應到 AFS 檔案空間頂層（/afs）的容體
    • root.cell，它是裝載在 cell 名稱 /afs 下的容體（例如，yourcompany.com cell 下的 /afs/yourcompany.com）

    由於您是在新的 cell 中架構第一個「AFS 伺服器」，因此必須在伺服器上建立 cell 的根容體，根容體會在伺服器架構期間被自動地建立。

  19. 選擇下一步按鈕。此時「複製」對話方塊會出現。

    如果您想要利用 AFS 複製功能的優點，就必須複製 AFS 根容體。 複製程序會建立一個或多個唯讀的 AFS 容體副本，並將這些副本分送到一個或多個其它網站（AFS 分割區和伺服器）。複製作業使 AFS 容體的內容得以被一部或多部「AFS 檔案伺服器」的機器存取，藉以提高系統的效率並增加資料的可用性。

    由於您是在新的 cell 中架構第一個「AFS 伺服器」，因此必須在伺服器上複製 cell 的根容體，根容體會在伺服器架構期間被自動地複製。

  20. 選擇下一步按鈕。「系統控制服務」對話方塊就會出現。

    「系統控制伺服器」會將新版的「AFS 伺服器」架構資訊分送到所有的 AFS 伺服器機器。 一般建議您將 AFS cell 中的第一個伺服器指定為「系統控制伺服器」。 （執行 AFS for Windows 國際版本的 cell 並不使用「系統控制伺服器」來分送系統架構檔。）
    註:之後若有需要，可將「系統控制伺服器」的職務指定給不同的伺服器機器。 然而，根據 cell 中 AFS 伺服器的數目而定，將職務指定給另一部機器的程序可能非常耗時。

  21. 若要將此「AFS 伺服器」架構為 AFS cell 的「系統控制伺服器」，請選擇架構為系統控制伺服器選項。如果您不要將此「AFS 伺服器」架構為此 AFS cell 的「系統控制伺服器」，請選擇不要架構為系統控制伺服器選項。
  22. 選擇下一步按鈕。此時「架構系統」對話方塊會出現。

    此時會顯示出一個步驟清單，該清單將用來架構此「AFS 伺服器」，讓您在開始實際的架構程序之前，先檢視這些步驟。
    註:若要返回前一個步驟以檢視或修改您的選擇，請選擇上一步按鈕。

  23. 若要開始在此機器上架構「AFS 伺服器」，請選擇架構按鈕。「AFS 伺服器」會根據您的設定來架構。 對話方塊底端的進度列指示在進行中的步驟。 然後會有一個訊息框出現，指出架構已完成。

將 AFS 伺服器架構到現有的 AFS cell 中：

  1. 從開始功能表中，選擇程式集，選擇 IBM AFS，然後選擇伺服器，接著選擇架構精靈。「AFS 伺服器快速啟動精靈」會開啟。
  2. 選擇下一步按鈕。「Cell 和伺服器資訊」對話方塊會出現。
  3. 選擇使這個主電腦成為現有的 AFS cell 中的一部伺服器選項。
  4. 在 Cell 名稱方塊中，輸入您要新增新的「AFS 伺服器」到其中之 AFS cell 的名稱。
  5. 選擇下一步按鈕。「管理資訊」對話方塊就會出現。
  6. 在名稱對話方塊中，輸入 AFS 管理帳戶的使用者名稱，例如 admin，或具有管理權限的 AFS 使用者帳戶名稱。
  7. 在密碼方塊中，輸入 AFS 管理帳戶的密碼，或輸入鍵入在名稱方塊中之具有管理權限的使用者帳戶的密碼。
  8. 在 AFS 伺服器方塊中，輸入此 AFS cell 中的一部執行「AFS 伺服器」的主電腦名稱。AFS 架構資訊會由伺服器中取出，以用於架構新的「AFS 伺服器」。
  9. 選擇下一步按鈕。「檔案服務」對話方塊就會出現。

    「AFS 檔案伺服器」會將所要求的檔案和資料從伺服器傳遞到「AFS 用戶端」。 「檔案伺服器」會儲存檔案和資料，處理複製、移動、建立，及刪除檔案和目錄的要求；追蹤伺服器上每一個檔案和目錄的狀態資訊。

    若要將此「AFS 伺服器」架構為「檔案伺服器」，請選擇是，架構為檔案伺服器選項。如果您不要將此「AFS 伺服器」架構為「檔案伺服器」，請選擇否，不要架構為檔案伺服器選項。

  10. 選擇下一步按鈕。「資料庫服務」對話方塊就會出現。

    每一個 AFS cell 都必須至少包含一個「資料庫伺服器」。 每一個「資料庫伺服器」都執行了維護 AFS 資料庫的資料庫程序：「鑑別資料庫」、「保護資料庫」 、「容體位置資料庫」，及選擇性的「備份資料庫」。

    若要將此「AFS 伺服器」架構為「資料庫伺服器」，請選擇是，架構為資料庫伺服器選項。如果您正在新增伺服器的 AFS cell 中已經有一部「系統控制伺服器」，您可以在「系統控制伺服器」方塊中輸入其主電腦名稱。 此伺服器將會更新 AFS 架構資訊（例如，每一部 AFS 伺服器機器上的 CellServDB 檔案所維護的「AFS 資料庫伺服器」清單）。 如果您不要將此「AFS 伺服器」架構為「資料庫伺服器」，請選擇否，不要架構為資料庫伺服器選項。

  11. 選擇下一步按鈕。「備份伺服器」對話方塊就會出現。

    「備份伺服器」負責維護「備份資料庫」，其中儲存了與「備份」系統相關的資訊。 「備份伺服器」使得 AFS 系統管理員可以將 AFS 檔案空間中的資料從容體備份到磁帶中。 當發生資料從檔案系統中遺失的狀況時（例如，假設系統或磁碟故障而導致資料遺失），便可以從磁帶復置。
    註:「備份伺服器」只能架構在被架構為「資料庫伺服器」的機器上。 而且，如果「備份伺服器」被架構在 cell 中的任何「資料庫伺服器」上，則它也必須架構在 cell 中的所有「資料庫伺服器」上。

  12. 若要將此「AFS 伺服器」架構為「備份伺服器」，請選擇是，架構為備份伺服器選項。如果您不要將此「AFS 伺服器」架構為「備份伺服器」，請選擇否，不要架構為備份伺服器選項。
  13. 選擇下一步按鈕。「AFS 分割區」對話方塊就會出現。

    若要將此「AFS 伺服器」架構為「檔案伺服器」，您可以指定一個 NTFS 容體作為 AFS 分割區。 每一部「AFS 檔案伺服器」至少都必須有一個專門用來儲存 AFS 容體的分割區，而所有的 AFS 容體都必須常駐在已經被指定為 AFS 分割區的分割區上。 在 Windows NT 的機器上，只有 NTFS 容體可被指定為 AFS 分割區。 除此之外，AFS 分割區只能建立在空的（或只包含「Windows NT 資源回收筒」的磁碟分割區）NTFS 容體上。

    若要指定容體作為 AFS 分割區，請選擇是，建立分割區選項。在 NTFS 容體清單中，選擇您要指定為 AFS 分割區的容體。 在 AFS 分割區名稱方塊中，輸入分割區名稱的最後一個部分。
    註:一部「AFS 伺服器」上最多可存在有 256 個 AFS 分割區。 依照慣例，每個分割區都命名為 /vicepx，其中 x 是一個或兩個英文字母的小寫字母。 AFS 分割區可命名為 /vicepa、/vicepb，依此類推一直到 /vicepz。其它的分割區可命名為 /vicepaa 到 vicepaz，依此類推一直到 /vicepiv。

    強烈建議您使用 NTFS 容體的磁碟機代號作為分割區名稱的最後一個部分。

    如果您不要將容體指定為 AFS 分割區，請選擇否，不要建立分割區選項。

  14. 選擇下一步按鈕。「AFS Root 容體」對話方塊就會出現。

    AFS 根容體是兩個容體，每一個 AFS cell 都必須包含在其檔案系統中。 它們分別為：

    • root.afs，它是對應到 AFS 檔案空間頂層（/afs）的容體
    • root.cell，它是裝載在 cell 名稱 /afs 下的容體（例如，yourcompany.com cell 下的 /afs/yourcompany.com）
    因為您是將此「AFS 伺服器」新增到現有的 AFS cell 中，所以 AFS 根容體可能已經存在於 cell 中，且「AFS 伺服器 架構精靈」會指示您不需要建立根容體。
    註:如果因為某些原因，而使 AFS 根容體不存在於此 AFS cell 中，您可以選擇是，建立根容體選項，在此「AFS 伺服器」上建立根容體。
  15. 選擇下一步按鈕。此時「複製」對話方塊會出現。

    如果您想要體驗 AFS 複製功能的優點，就必須複製 AFS 根容體。 複製程序會建立一個或多個唯讀的 AFS 容體副本，並將這些副本分送到一個或多個其它網站（AFS 分割區和伺服器）。 複製作業使 AFS 容體的內容得以被一部或多部「AFS 檔案伺服器」的機器存取，藉以提高系統的效率並增加資料的可用性。

    因為您是將此「AFS 伺服器」新增到現有的 AFS cell 中，所以 AFS 根容體可能已經被複製，且「AFS 伺服器架構精靈」會指示您不需要複製 AFS 根容體。
    註:如果因為某些原因，而使 AFS 根容體沒有在此 AFS cell 中被複製，您可以選擇是，複製根容體選項，在此「AFS 伺服器」上複製 AFS cell 的根容體。

  16. 選擇下一步按鈕。「系統控制服務」對話方塊就會出現。

    如果 cell 是執行試用版的 AFS for Windows，則「系統控制伺服器」會將新版的「AFS 伺服器」架構資訊分送到所有的 AFS 伺服器，而「系統控制用戶端」機器會從「系統控制」機器取得常用的 AFS 架構檔。（執行 AFS for Windows 國際版本的 cell 並不使用「系統控制伺服器」來分送系統架構檔，而「系統控制用戶端」也不會取得這些檔案。）

  17. 若要將此「AFS 伺服器」架構為 AFS cell 的「系統控制伺服器」， 請選擇架構為系統控制伺服器選項。若要將此「AFS 伺服器」架構為「系統控制用戶端」，請選擇架構為系統控制用戶端選項，並輸入此 AFS cell 中「系統控制伺服器」的主電腦名稱。此「AFS 伺服器」將會從所指定的伺服器取得新版的「AFS 伺服器」架構資訊。 如果您不要將此「AFS 伺服器」架構為此 AFS cell 的「系統控制伺服器」或「系統控制用戶端」，請選不要架構為系統控制用戶端或伺服器選項。
  18. 選擇下一步按鈕。此時「架構系統」對話方塊會出現。

    此時會顯示出一個步驟清單，該清單將用來架構此「AFS 伺服器」，讓您在開始實際的架構程序之前，先檢視這些步驟。
    註:若要返回前一個步驟以檢視或修改您的選擇，請選擇上一步按鈕。

  19. 若要開始在此機器上架構「AFS 伺服器」，請選擇架構按鈕。如果您是將「AFS 伺服器」架構到一個 AFS cell 中，而該 cell 中的「資料庫伺服器」執行比 3.5 版還舊的 AFS，則會出現一個對話方塊，提示您輸入 AFS principal 密碼。

    「AFS 伺服器」會根據您的設定來架構。 對話方塊底端的進度列指示在進行中的步驟。 然後會有一個訊息框出現，指出架構已完成。

架構 AFS 控制中心

註:如果您已經結合了「AFS 伺服器」或「AFS 用戶端」（或兩者皆有）來安裝「AFS 控制中心」，那麼就不需要架構「AFS 控制中心」。「AFS 控制中心」會在「AFS 伺服器」或「AFS 用戶端」架構時，自動地架構。如果您只安裝了「AFS 控制中心」，則必須先在系統上架構「控制中心」，才能使用它。
  1. 從開始功能表中，選擇設定，然後選擇控制台。
  2. 按兩下 AFS 控制中心圖示。「AFS 控制中心內容」 對話方塊會出現。
  3. 在預設 Cell 方塊中，輸入預設要管理之 AFS cell 的完整名稱。
  4. 如果「AFS 控制中心」要管理的 cell 沒有出現在 AFS cell 清單中，請選擇新增按鈕。「新增 Cell」對話方塊會開啟。在 AFS Cell 方塊中輸入 cell 名稱，並在說明方塊中輸入簡短的說明。

    選擇新增按鈕。「新增伺服器」對話方塊會開啟。在伺服器名稱方塊中，自所選取的 cell 內輸入「容體位置伺服器」的名稱。 選擇確定關閉「新增伺服器」對話方塊。 重複此程序，加入 cell 中所有「容體位置伺服器」的資訊。在輸入完所有的伺服器資訊之後，選擇確定關閉「新增 Cell」對話方塊。

  5. 選擇確定關閉「AFS 控制中心內容」對話方塊。

「AFS 控制中心」現在已經架構好了。

解除安裝 AFS for Windows

本節指出解除安裝的必備需求，提供解除安裝 AFS for Windows 的指示，並列出解除安裝程序對您的系統所做的變更。

重新安裝與升級

在 Windows NT 機器上，重新安裝或升級軟體之前並不需要解除安裝 AFS for Windows 的元件。若要重新安裝 AFS for Windows，請遵循安裝 AFS for Windows 中說明的安裝程序進行。 在安裝程序期間，早期版本的 AFS 元件會被取代。 AFS 架構資訊會被保留。

在 Windows 95 或 Windows 98 的機器上，您必須先解除安裝之前安裝的 AFS Light 元件（如 解除安裝 AFS for Windows 中的說明），才能重新安裝或升級 AFS Light。

解除安裝必備需求

解除安裝 AFS 會導致所有的 AFS 應用程式檔案被刪除。 如果有其它應用程式正在使用這些檔案，則它們便無法被刪除。基於這個理由，您必須在重新安裝 AFS for Windows 之前，先關閉所有的 AFS 對話方塊。

如果您是為了不使用這部機器才解除安裝「AFS 伺服器」，以下的必備需求可避免資料的流失：

  1. 如果「AFS 伺服器」是作為「檔案伺服器」使用，請將所有的讀取/寫入容體搬移到另一部「AFS 檔案伺服器」，並移除所有唯讀的容體。
  2. 取消架構「AFS 伺服器」。開啟「AFS 伺服器 架構」公用程式，並選擇伺服器標籤。清除所有的勾選框，並選擇確定。

解除安裝 AFS for Windows

  1. 從開始功能表中，選擇設定，然後選擇控制台。
  2. 按兩下新增/移除程式圖示。「新增/移除程式內容」對話方塊會出現，並顯示出安裝/移除標籤。
  3. 關閉「控制台」。
  4. 選取要解除安裝的 AFS 元件，並選取新增/移除按鈕。「確認刪除檔案」對話方塊會出現，以確認您是否真的要移除所選取的 AFS for Windows 元件。按一下是繼續解除安裝程序。
  5. 此時 AFS 訊息框會出現，詢問您是否要保留架構資訊。選擇是保留架構資訊，或選擇否刪除所有的架構資訊。（沒有架構資訊與「AFS 補充文件」元件連結。如果您是從系統移除此元件，則 AFS 訊息框不會出現。）
  6. 「從電腦移除程式」對話方塊會開啟，並顯示出將要從系統移除的元件。
    註:此時可能會出現一個訊息框，詢問您是否要移除不再被其它元件使用的共用 AFS 檔案。按一下全部皆是，完全地移除所選取的 AFS 元件。

所選取的 AFS for Windows 元件現在已經被移除了。如果您安裝了整組的 AFS for Windows 元件，則必須重複步驟 4-6 來個別移除每一個元件。

對系統所做的變更

解除安裝 AFS 用戶端對系統所做的變更

解除安裝「AFS 用戶端」會對系統做下列的變更：

  • 從 \Program Files\Ibm\Afs\Client\Program 目錄移除所有的「AFS 用戶端」檔案，移除 Client 目錄，如果沒有安裝其它的 AFS 元件，則移除 Ibm 目錄。
    註:如果該目錄中包含了 AFS for Windows 安裝程式所安裝之檔案以外的檔案，則這些目錄就不會被移除。
  • 如果沒有安裝其它的 AFS 元件，則從開始功能表中移除 IBM AFS 程式群組。
  • 從「控制台」移除 AFS 用戶端架構圖示。
  • 將「AFS 功能表」從「Windows NT 檔案總管」的上下文功能表中移除。
  • 刪除 IBM AFS 用戶端 服務。
  • 從系統移除「AFS 用戶端」相關的登錄項目。請注意：如果您選擇了保留架構資訊，則在解除安裝程序結束後，仍會有一些資訊留在登錄中。

解除安裝 AFS Light 對系統所做的變更

解除安裝 AFS Light 會對系統做下列的變更：

  • 從 \Program Files\Ibm\Afs\Client\Program 目錄移除所有的 AFS 檔案，並移除 Ibm 目錄。
    註:如果該目錄中包含了 AFS for Windows 安裝程式所安裝之檔案以外的檔案，則這些目錄就不會被移除。
  • 從開始功能表中移除 IBM AFS 程式群組。
  • 從「控制台」移除 AFS Light 架構圖示。
  • 將「AFS 功能表」從「Windows 檔案總管」的上下文功能表中移除。
  • 從系統移除 AFS Light 相關的登錄項目。 請注意：如果您選擇了保留架構資訊，則在解除安裝程序結束後，仍會有一些資訊留在登錄中。

解除安裝 AFS 伺服器對系統所做的變更

解除安裝「AFS 伺服器」會對系統做下列的變更：

  • 從 \Program Files\Ibm\Afs\Server 目錄移除所有的「AFS 伺服器」檔案，移除 Server 目錄，如果沒有安裝其它的 AFS 元件，則移除 Ibm 目錄。
    註:如果該目錄中包含了 AFS for Windows 安裝程式所安裝之檔案以外的檔案，則這些目錄就不會被移除。 另外，如果您選擇了保留架構資訊，則 \Program Files\Ibm\Afs\Server 目錄下的某些檔案不會被移除。
  • 如果沒有安裝其它的 AFS 元件，則從開始功能表中移除 IBM AFS 程式群組。
  • 從「控制台」移除 AFS 伺服器架構圖示。
  • 刪除 IBM AFS 伺服器服務。
  • 從系統移除「AFS 伺服器」相關的登錄項目。 請注意：如果您選擇了保留架構資訊，則在解除安裝程序結束後，仍會有一些資訊留在登錄中。

解除安裝 AFS 控制中心對系統所做的變更

解除安裝「AFS 控制中心」會對系統做下列的變更：

  • 從 \Program Files\Ibm\Afs\Control Center 目錄中移除所有的「AFS 控制中心」檔案，移除 Control Center 目錄，如果沒有安裝其它的 AFS 元件，則移除 Ibm 目錄。
    註:如果該目錄中包含了 AFS for Windows 安裝程式所安裝之檔案以外的檔案，則這些目錄就不會被移除。
  • 如果沒有安裝其它的 AFS 元件，則從開始功能表中移除 IBM AFS 程式群組。
  • 從「控制台」移除 AFS 控制中心圖示。 請注意：這個圖示只有在系統上沒有安裝其它的 AFS for Windows 元件時，才會出現在「控制台」中。
  • 從系統移除「AFS 控制中心」相關的登錄項目。 請注意：如果您選擇了保留架構資訊，則在解除安裝程序結束後，仍會有一些資訊留在登錄中。

解除安裝 AFS 補充文件對系統所做的變更

解除安裝 AFS 補充文件會對系統做下列的變更：

  • 從 \Program Files\Ibm\Afs\Documentation\Html 目錄移除 SysAdminGd 目錄及 CmdRef 目錄，如果沒有安裝其它的 AFS 元件，則移除 Ibm 目錄。
    註:如果該目錄中包含了 AFS for Windows 安裝程式所安裝之檔案以外的檔案，則這些目錄就不會被移除。
  • 如果沒有安裝其它的 AFS 元件，則從開始功能表中移除 IBM AFS 程式群組。
  • 從系統移除 AFS 補充文件相關的登錄項目。
