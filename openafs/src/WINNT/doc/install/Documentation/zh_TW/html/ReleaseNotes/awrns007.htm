AFS 控制中心版本注意事項

簡介

本節包含 AFS for Windows 3.6 版之 AFS 控制中心元件的重要資訊。 AFS 控制中心提供兩個功能強大的圖形使用者介面（GUI）工具， 以協助 AFS 系統管理者執行 AFS Cell 管理：AFS 伺服器管理程式和 AFS 帳戶管理程式。

AFS 控制中心安裝基本需求

您的系統必須符合下列硬體、軟體及管理權限基本需求， 才能安裝 AFS for Windows 的 AFS 控制中心元件。

硬體

安裝 AFS 控制中心的硬體基本需求：

  • 80486/66 MHz（或更快）微處理器或 Pentium 等級微處理器的 PC
  • 至少 16 MB 的 RAM
  • 至少 4 MB 可用磁碟空間的硬碟

軟體

安裝 AFS 控制中心的軟體基本需求：

  • Microsoft Windows NT 4.0 加上 Service Pack 4 或 Service Pack 5

管理權限基本需求

您必須是 Windows 系統的本端管理者群組成員之一， 才能夠安裝 AFS 控制中心。

產品注意事項

本節強調顯示關於 AFS for Windows 3.6 版之 AFS 控制中心元件重要的一般資訊。

必須將 AFS 控制中心安裝在其路徑只含 ANSI 字元的目錄中

必須將 AFS for Windows 軟體安裝在其路徑只含 ANSI 字元的目錄中， 例如預設目錄 C:\Program Files。

暫存檔

Windows 版 AFS 控制中心偶而會建立暫存檔。 若要控制這些檔案的建立位置，請將環境變數（TMP 或 TEMP）設定為您所要之暫時目錄的完整路徑。 若不指定暫時目錄， 則任何暫存檔將在程序的現行工作目錄中建立。

AFS 控制中心的國家語言支援

AFS 提供有關 AFS 控制中心圖形使用者介面（GUI）工具和說明文件的國家語言支援， 包括雙向 script 的支援（希伯來、阿拉伯等）。安裝的語言字串係由機器的系統預設語言環境所決定， 亦即「控制台」的「國別設定內容」對話方塊中所指定的。
註:機器的系統預設語言環境可能不同於 使用者語言環境（由使用者指定）。不過，AFS for Windows 所安裝的語言字串，一律由系統預設語言環境來決定。

若目前的語言環境沒有可用的語言字串， 則依預設值安裝英語字串。

Windows 系統不使用 kpwvalid 密碼強化檢查器

從 Windows 系統變更 AFS 使用者密碼時，不使用kpwvalid 程式來檢查新密碼的品質。 （在 UNIX 系統上，若 kpwvalid 程式和 kpasswd 程式位於相同目錄中， 則由 kpwvalid 程式檢查新密碼使用者的品質。）

控制中心應用程式所取得的記號於 25 小時以後失效

為了使用 AFS 伺服器管理程式或 AFS 帳戶管理程式， 使用者必須取得 AFS 記號。不論使用者指定的「最大通行證使用期限」多長，在任一應用程式中取得的記號，使用期限一律為 25 小時又 25 分鐘。

伺服器管理程式顯示「容體位置資料庫」（VLDB）中所參照的全部伺服器登錄

AFS 伺服器管理程式顯示「容體位置資料庫」（VLDB）中每一個伺服器登錄的伺服器圖示。 若 AFS 伺服器已卸除任務， 但尚未移除其 VLDB 登錄，則「伺服器管理程式」會繼續顯示伺服器的圖示。

若要在執行 AFS 3.5 版或更高版本的「資料庫伺服器」上，從 VLDB 中移除不用的 AFS 檔案伺服器登錄， 請發出下列指令： 

vos changeaddr <ip_address> -remove

其中，ip_address 是已卸除任務之伺服器機器的 網際網路通信協定（IP）位址。

在執行 AFS 3.5 版以前的 AFS 資料庫伺服器上， 目前沒有指令可從 VLDB 中移除不用的伺服器登錄。 不過，若要防止「AFS 伺服器管理程式」顯示已卸除任務的伺服器， 您可修改 Windows NT 登錄，讓「AFS 伺服器管理程式」略過特定 IP 位址型樣的所有機器。

若要防止「AFS 伺服器管理程式」顯示特定 IP 位址型樣的伺服器機器， 請使用下列步驟來修改 Windows NT 登錄：

  1. 從「開始」功能表中，選擇執行。在「開啟」方塊中， 鍵入 regedit，然後選擇確定。開啟「Windows NT 登錄編輯器」。
  2. 在 \\HKEY_LOCAL_MACHINE\Software\TransarcCorporation\AFS Control Center 機碼上按一下滑鼠右鍵。從關聯式功能表中， 選擇新增，然後選擇 DWORD 值。 該機碼會新增一個「新數值」登錄。
  3. 鍵入 IgnoreBadAddrs 做為新數值的名稱。
  4. 按兩下新的 IgnoreBadAddrs 登錄。出現「編輯 DWORD 數值」對話方塊。
  5. 在數值資料方塊中，以十六進位表示法， 指定要忽略的 IP 位址或 IP 位址範圍。例如，若要讓 「AFS 伺服器管理程式」略過 VLDB 中 IP 位址符合 10.0.0.* 型樣的任何機器， 請在數值資料方塊中輸入 0A0000FF。（十六進位值 FF 代表一個萬用字元。）
  6. 選取確定，關閉「編輯 DWORD 數值」方塊， 儲存新的登錄項目。
  7. 關閉「登錄編輯器」。

註:為了善用此功能，建議您將 VLDB 中任何已卸除任務之伺服器的 IP 位址， 變更為特定範圍內易於辨識的未用 IP 位址，例如 10.0.0.1、 10.0.0.2、10.0.0.3 等。 然後，依上述步驟新增 IgnoreBadAddrs 登錄項目， 讓「AFS 伺服器管理程式」略過特定範圍內的全部 IP 位址。

限制

AFS for Windows 3.6 版的 AFS 控制中心元件，目前沒有已知的限制。
