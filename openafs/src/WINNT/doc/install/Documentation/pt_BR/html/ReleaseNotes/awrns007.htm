Notas de Release do AFS Control Center

Introdução

Esta seção contém informações importantes sobre o componente do AFS Control Center do AFS para Windows, versão 3.6. O AFS Control Center fornece duas poderosas ferramentas de interfaces gráficas de usuário (GUI) para assistir os administradores do sistema AFS na administração da célula AFS: O Gerenciador de Servidor do AFS e o Gerenciador de Contas do AFS.

Requisitos de Instalação do AFS Control Center

O sistema deve atender aos seguintes requisitos de hardware, software e privilégios administrativos para instalar o componente do AFS Control Center do AFS para Windows.

Hardware

Os requisitos de hardware para instalar o AFS Control Center são:

  • Um PC usando um microprocessador 80486/66 MHz (ou mais rápido) ou um microprocessador baseado em Pentium
  • No mínimo 16 MB de RAM
  • Um disco rígido com no mínimo 4 MB de espaço disponível em disco

Software

Os requisitos de software para instalar o AFS Control Center são:

  • Microsoft Windows NT 4.0 com Service Pack 4 ou Service Pack 5

Requisito de Privilégio Administrativo

É preciso ser um membro do grupo local Administradores no sistema Windows para instalar o AFS Control Center.

Notas do Produto

Esta seção destaca informações gerais importantes sobre o componente do AFS Control Center do AFS para Windows, versão 3.6.

O AFS Control Center Deve Ser Instalado em um diretório Cujo Caminho Contenha Somente Caracteres ANSI

O software do AFS para Windows somente pode ser instalado em um diretório cujo caminho contenha somente caracteres ANSI, por exemplo o diretório padrão C:\Arquivos de Programas.

Arquivos Temporários

O AFS Control Center para Windows ocasionalmente cria arquivos temporários. Para controlar onde tais arquivos são criados, defina a variável de ambiente (TMP ou TEMP) para o nome do caminho completo do diretório temporário preferencial. Se um diretório temporário não for especificado, quaisquer arquivos temporários serão criados no diretório de trabalho atual do processo que criar os arquivos.

Suporte a Idioma Nacional para o AFS Control Center

O AFS fornece suporte a idioma nacional para as ferramentas e documentação da interface gráfica com o usuário (GUI) do AFS Control Center, incluindo suporte a scripts bidirecionais (Hebraico, Árabe, etc.). As cadeias de idioma instaladas são determinadas pelas características locais padrão da máquina, conforme especificadas na caixa de diálogo Propriedades de Configurações Regionais.
Nota:As características locais padrão do sistema para uma máquina podem ser diferentes das características locais de usuário (especificadas por um usuário) na máquina. No entanto, as cadeias de idioma instaladas com o AFS para Windows são sempre determinadas pelas características locais padrão do sistema.

Se não existirem cadeias de idioma para as características locais atuais, as cadeias de idioma para Inglês serão instaladas por padrão.

O Verificador de Força da Senha kpwvalid Não É Usado em Sistemas Windows

Quando uma senha de usuário do AFS é alterada a partir de um sistema Windows, o programa kpwvalid não é usado para verificar a qualidade da nova senha. (Em sistemas UNIX, se existir um programa kpwvalid no mesmo diretório do programa kpasswd, o programa kpwvalid verifica a qualidade de toda nova senha de usuário.)

Tokens Obtidos para Aplicações do Centro de Controle Vencem após 25 Horas

Para usar o Gerenciador de Servidor do AFS ou o Gerenciador de Contas do AFS, os usuários devem obter tokens do AFS. O tempo de operação dos tokens obtidos nas duas aplicações é sempre de 25 horas e 25 minutos, independente do Tempo Máximo da Operação do Ticket designado para o usuário.

O Gerenciador de Servidor Exibe Todas as Entradas de Servidor Referenciadas no Banco de Dados de Localização de Volumes (VLDB)

O Gerenciador de Servidor do AFS exibe um ícone de servidor para cada entrada de servidor no Banco de Dados de Localização de Volumes (VLDB). Se um servidor AFS foi desativado, mas sua entrada no VLDB não foi removida, o Gerenciador de Servidor continuará a exibir um ícone para o servidor.

Para remover uma entrada obsoleta de Servidor de Arquivos AFS do VLDB em um Servidor de Banco de Dados executando o AFS versão 3.5 ou posterior, emita o seguinte comando: 

vos changeaddr <endereço_ip> -remove

onde endereço_ip é o endereço Internet Protocol (IP) da máquina do servidor que foi desativado.

Em Servidores de Bancos de Dados AFS executando uma versão do AFS anterior à versão 3.5, não existe atualmente um comando para remover entradas obsoletas de servidores no VLDB. Entretanto, para evitar que o Gerenciador de Servidor do AFS exiba servidores desativados, pode-se modificar o Registro do Windows NT de modo que o Gerenciador de Servidor do AFS ignore todas as máquinas com endereços IP de um padrão especificado.

Para evitar que o Gerenciador de Servidor do AFS exiba máquinas de servidores com endereços IP de um padrão especificado, use as seguintes etapas para modificar o registro do Windows NT:

  1. A partir do menu Iniciar, escolha Executar. Na caixa Abrir, digite regedit, e em seguida escolha OK. O Editor do Registro do Windows NT é aberto.
  2. Clique o botão direito na chave \\HKEY_LOCAL_MACHINE\Software\TransarcCorporation\AFS Control Center. No menu de contexto que aparece, escolha Novo, e em seguida escolha Valor DWORD. Uma entrada de Novo Valor é incluída na chave.
  3. Especifique o nome da entrada de novo valor digitando IgnoreBadAddrs como o novo valor.
  4. Dê um clique duplo na nova entrada IgnoreBadAddrs. A caixa de diálogo Editar Valor DWORD aparece.
  5. Na caixa Dados do valor, especifique em notação hexadecimal o endereço IP ou caixa de endereços IP a ignorar. Por exemplo, para instruir o Gerenciador de Servidor do AFS a ignorar quaisquer máquinas listadas no VLDB com endereços IP correspondendo ao padrão 10.0.0.*, digite 0A0000FF na caixa Dados do valor. (O valor hexadecimal FF representa um caractere curinga.)
  6. Escolha OK para fechar a caixa Editar valor DWORD e salvar a nova entrada no registro.
  7. Feche o Editor do Registro.

Nota:Para obter vantagem deste recurso, é recomendável alterar os endereços IP de quaisquer servidores desativados no VLDB para endereços IP não utilizados facilmente reconhecíveis em uma faixa específica, por exemplo 10.0.0.1, 10.0.0.2, 10.0.0.3 etc. Em seguida, inclua a entrada de registro IgnoreBadAddrs conforme descrito nas etapas anteriores para instruir o Gerenciador de Servidor do AFS a ignorar todos os endereços IP dentro da faixa especificada.

Limitações e Restrições

Não existem atualmente limitações nem restrições conhecidas para o componente do AFS Control Center do AFS para Windows, versão 3.6.
