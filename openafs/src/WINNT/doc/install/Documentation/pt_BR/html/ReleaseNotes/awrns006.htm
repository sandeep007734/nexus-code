Notas de Release do AFS Server

Introdução

Esta seção contém informações importantes sobre o componente AFS Server do AFS para Windows, versão 3.6. O AFS Server executa processos de servidor do AFS em uma máquina Windows NT. Um AFS Server em uma máquina Windows NT pode ser configurado como um ou mais dos seguintes: um Servidor de Arquivos AFS, um Servidor de Banco de Dados AFS e um Servidor de Controle do Sistema AFS.

Requisitos de Instalação do AFS Server

O sistema deve atender aos seguintes requisitos de hardware, software e privilégios administrativos para instalar o componente AFS Server do AFS para Windows.

Hardware

Os requisitos de hardware para instalar o AFS Server são:

  • Um PC usando um microprocessador 80486/66 MHz (ou mais rápido) ou um microprocessador baseado em Pentium

    Recomendado: Um PC usando um microprocessador Pentium ou maior

  • No mínimo 16 MB de RAM

    Recomendado: 32 MB de RAM ou mais

  • Um disco rígido com no mínimo 20 MB de espaço disponível em disco
  • No mínimo um volume de disco NTFS para conter os dados do sistema de arquivos do AFS. O volume NTFS não deve conter dados ou arquivos a não ser os de uma Lixeira do Windows NT para ser elegível para configuração como uma partição AFS.

Software

Os requisitos de software para instalar o AFS Server são:

  • Microsoft Windows NT 4.0 com Service Pack 4 ou Service Pack 5

Requisito de Privilégio Administrativo

É preciso ser um membro do grupo local Administradores no sistema Windows para instalar, configurar e iniciar o AFS Server.

Notas do Produto

Esta seção destaca informações gerais importantes sobre o componente AFS Server do AFS para Windows, versão 3.6. Ela inclui descrições de novos recursos e aperfeiçoamentos disponibilizados neste release do AFS Server.

Suporte para Volumes Maiores Que 2 GB

Um volume AFS é uma coleção de arquivos relacionados que são organizados em uma unidade única, facilmente gerenciável, e que podem ser movidos facilmente entre máquinas de Servidores de Arquivos para facilitar o balanceamento de carga na rede. Cada volume AFS tem uma quota específica associada a ele. A quota de um volume especifica a quantidade máxima de espaço em disco que as informações no volume podem ocupar. A quota é definida em uma base por volume, de modo que pode ser aumentada para volumes que contêm mais dados e diminuída para volumes que não necessitem do espaço adicional. O AFS para Windows, versão 3.6 agora suporta volumes com quotas maiores que 2 GB.

Rotinas de Criptografia Usadas pelo Servidor de Controle do Sistema Estão Incluídas com Todas as Versões do AFS para Windows

As rotinas de criptografia usadas pelo Serviço de Atualização na máquina do Servidor de Controle do sistema agora estão sendo incluídas com todas as versões do AFS para Windows devido ao relaxamento dos regulamentos de exportação do governo americano. Anteriormente, regulamentos do governo proibiam a exportação desta rotinas de criptografia. Assim, locais que não estivessem executando a distribuição para o Estados Unidos do AFS não podiam usar um Servidor de Controle do Sistema para distribuir arquivos de configuração, porque isso permitiria a transmissão sem criptografia de informações sensíveis do sistema. Agora, todos os locais podem usar um servidor de Controle do Sistema para distribuir novas versões de informações de configuração do AFS Server para todas as máquinas de Servidores de Arquivos AFS.

O Nome do Computador do AFS Server Deve Corresponder ao Nome do Host do Serviço de Nomes

O nome do computador de uma máquina de AFS Server (conforme exibido no campo Nome do Computador na guia Identificação da caixa de diálogo Rede) deve corresponder ao nome do host atribuído ao computador pelo serviço de nomes (normalmente o Domain Name Service, ou DNS) usado para mapear nomes para endereços do Internet Protocol (IP). Por exemplo, se a máquina na qual o AFS Server está instalado tiver o nome de computador afsserver1 e estiver no domínio yourcompany.com, a entrada correspondente no DNS para a máquina deve ser afsserver1.yourcompany.com.

Sincronização dos Relógios de Tempo

Para suportar protocolos de segurança e de replicação de dados, os relógios de tempo de todas as máquinas executando servidores AFS devem estar sincronizados. O software de sincronização de tempo pode ser obtido de diversas fontes; por exemplo, a Microsoft distribui o serviço de sincronização de tempo timeserv com o Kit de Recursos do Servidor Windows NT.

Segurança de Máquinas do AFS Server

Se a máquina na qual o software do AFS Server está instalado não for segura (isto é, se usuário não administrativo puder efetuar o logon na máquina), é altamente recomendável que os diretórios do software de AFS Server e todas as partições AFS na máquina sejam tornados seguros conforme apropriado para o ambiente e tipo do sistema de arquivos. Entretanto, ao tornar seguras as máquinas do AFS Server, assegure-se de que o proprietário local SYSTEM do Windows NT tenha acesso total ao software do AFS e a todas as partições AFS; caso contrário, os processos do AFS não poderão ser executados.

Encerramento de Máquinas de Servidores de Arquivos AFS

Antes de encerrar uma máquina do Windows NT que esteja executando como um Servidor de Arquivos AFS, sempre pare o Servidor de Arquivos AFS na máquina.

Quando o Servidor de Arquivos AFS é iniciado em uma máquina, o processo cria um arquivo de recuperação. Quando o Servidor de Arquivos é parado manualmente, esse arquivo de recuperação é removido. Entretanto, se uma máquina Windows NT for encerrada enquanto o Servidor de Arquivos AFS ainda estiver em execução, o arquivo de recuperação não será removido. Quando a máquina for reiniciada posteriormente, o Servidor de Arquivos iniciará automaticamente e, ao notar a presença do arquivo de recuperação, executará o Salvager. Como executar o Salvager pode tomar muito tempo, é sempre preferível parar o Servidor de Arquivos AFS em uma máquina antes de encerrá-la para evitar a execução desnecessária do processo Salvager.

Use um dos seguintes procedimentos para encerrar o Servidor de Arquivos AFS em uma máquina Windows NT.

  • No Painel de Controle do Windows NT:
    1. Dê um clique duplo no ícone Serviços. A caixa de diálogo Serviços aparece.
    2. Na lista Serviço, selecione o serviço IBM AFS Server.
    3. Escolha o botão Parar. Aparece uma caixa de mensagens, pedindo a confirmação de que deseja parar o serviço. Escolha o botão Sim.

      O Servidor de Arquivos AFS na máquina é parado.

  • No Prompt de Comandos do Windows NT:
    1. Digite o seguinte comando: 
      bos shutdown -server nome_da_máquina -wait

      onde nome_da_máquina é o nome da máquina do Windows NT na qual o Servidor de Arquivos AFS está executando.

    2. Pressione <Enter> para executar o comando.
Nota:Em geral, máquinas que estão executando como Servidores de Arquivos AFS são encerradas somente para efetuar manutenção preventiva.

Obtenção de Arquivos de Descarga de Desastre em Caso de Problema de Servidor

É recomendado que se configure todas as máquinas de AFS Server para criar um arquivo de descarga de desastre no caso de um defeito no servidor. Esta informação é importante para uso do pessoal de suporte técnico no diagnóstico de um problema no servidor. Consulte a documentação do Microsoft Windows NT para obter informações sobre como configurar o utilitário Dr. Watson para gerar um arquivo binário de descarga de desastre automaticamente quando ocorrer um erro de aplicação.

O AFS Server Deve Ser Instalado em um diretório Cujo Caminho Contenha Somente Caracteres ANSI

O software do AFS para Windows somente pode ser instalado em um diretório cujo caminho contenha somente caracteres ANSI, por exemplo o diretório padrão C:\Arquivos de Programas.

Arquivos Temporários

O AFS Server para Windows ocasionalmente cria arquivos temporários. Para controlar onde tais arquivos são criados, defina a variável de ambiente (TMP ou TEMP) para o nome do caminho completo do diretório temporário preferencial. Se um diretório temporário não for especificado, quaisquer arquivos temporários serão criados no diretório de trabalho atual do processo que criar os arquivos.

Suporte a Idioma Nacional para o AFS Server

O AFS fornece suporte a idioma nacional para as ferramentas e documentação da interface gráfica com o usuário (GUI) do AFS Server, incluindo suporte a scripts bidirecionais (Hebraico, Árabe, etc.). As cadeias de idioma instaladas são determinadas pelas características locais padrão da máquina, conforme especificadas na caixa de diálogo Propriedades de Configurações Regionais. Se não existirem cadeias de idioma para as características locais atuais, as cadeias de idioma para Inglês serão instaladas por padrão.
Nota:As características locais padrão do sistema para uma máquina podem ser diferentes das características locais de usuário (especificadas por um usuário) na máquina. No entanto, as cadeias de idioma instaladas com o AFS para Windows são sempre determinadas pelas características locais padrão do sistema.

O Verificador de Força da Senha kpwvalid Não É Usado em Sistemas Windows

Quando uma senha de usuário do AFS é alterada a partir de um sistema Windows, o programa kpwvalid não é usado para verificar a qualidade da nova senha. (Em sistemas UNIX, se existir um programa kpwvalid no mesmo diretório do programa kpasswd, o programa kpwvalid verifica a qualidade de toda nova senha de usuário.)

Configuração de um Servidor em uma Célula com Servidores de Bancos de Dados pré-3.5 Exige a Senha do Proprietário

Se configurar o AFS Server para Windows, versão 3.6, em uma célula AFS na qual os Servidores de Bancos de Dados estiverem executando uma versão do AFS anterior à versão 3.5, durante o processo de configuração uma caixa de diálogo solicita a senha do proprietário AFS.

Nota:Se por algum motivo a senha do proprietário AFS for desconhecida, pode-se digitar em vez dela a cadeia de 24 caracteres de dígitos octais representando a chave do proprietário AFS. Para obter a chave octal, emita o comando kas examine ou o comando boslistkeys em uma máquina de Servidor de Banco de Dados AFS. Note, o servidor deve estar executando em modo noauth para poder exibir a chave octal do proprietário AFS.

Limitações e Restrições

Esta seção descreve resumidamente as limitações e restrições conhecidas no componente AFS Server do AFS para Windows, versão 3.6.

O Processo bosserver do AFS Não Executa o Programa Notificador Quando um Processo Termina

O processo bosserver do AFS atualmente não executa o comando de notificador registrado (se for especificado um) quando um processo sob seu controle termina.
